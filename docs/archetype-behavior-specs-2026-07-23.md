# Детальные спеки поведения per-архетип («задача-первым», уровень 2)

Дата: 2026-07-23. Родитель: `archetype-first-screen-concept-2026-07-23.md`
(принципы П1–П5 и волны утверждены владельцем). Здесь — ЛОГИКА И ПОВЕДЕНИЕ
каждого архетипа: первый экран поэлементно, шаги funnel'а, крайние состояния,
пост-покупка, и **вопросы на согласование с дефолтами** (ID: F/H/MB/W/HW/E/R/RT).
Правило: вопрос без ответа владельца = действует дефолт.

## Волна 1

### FRISEUR (флагман, пример владельца)

**Первый экран (сверху вниз):**
1. Шапка: лого/название · меню · язык · пилюля «Jetzt erreichbar» (presence).
2. H1-зона: название + рейтинг ⌀·N (клик → отзывы) + адрес кратко + open-status.
3. **Primary-виджет «Termin buchen»:** чипы категорий услуг → список услуг
   (название · длительность · цена) с кнопкой «Buchen» у каждой.
4. Вторая кнопка: «Nicht sicher? → Finder» (П5).
5. Мобайл: то же + липкий buybar «Termin buchen».

**Поведение (funnel):**
1. «Buchen» у услуги → **шаг «Мастер» (G2, НОВОЕ):** карточки мастеров из
   staff-профилей A3 (фото · имя · специализация) + опция **«Egal —
   nächster freier Termin» ПЕРВОЙ**. 0–1 мастер → шаг пропускается.
2. Шаг «Слот»: неделя-календарь доступных слотов, отфильтрованных по мастеру;
   ближайший подсвечен.
3. Шаг «Контакт»: имя + телефон/email (+заметка). Без аккаунта.
4. Подтверждение: экран + письмо (перенос/отмена-ссылки; wa.me у видео-услуг).

**Состояния:** нет слотов на неделе → «Nächster freier Termin: {дата}» +
переход к той неделе + предложение другого мастера · всё закрыто (отпуск) →
«Rückruf anfordern» (тред inbox) · услуга с `primary_action=request` → заявка
вместо слота.

**Пост-покупка:** напоминание за 24ч ✅ · post-visit отзыв-письмо ✅ ·
win-back B4 ✅.

**Вопросы F (дефолты):**
- **F-1**: шаг «Мастер» отдельным экраном с «Egal»-опцией первой (дефолт) —
  или выбор мастера прямо на карточке услуги?
- **F-2**: вариативная цена услуги → показывать «ab X €» (дефолт: да).
- **F-3**: депозит при записи — дефолт: БЕЗ депозита (вкл. опцией владельца).

### HOTEL (флагман)

**Первый экран:** фото отеля фоном; поверх — название + рейтинг + локация;
**date-search виджет** (Anreise/Abreise-календарь · гости взрослые/дети ·
«Verfügbarkeit prüfen»); под ним карточки номеров «ab X €/Nacht» (фото ·
вместимость · 2-3 удобства).

**Поведение:**
1. Даты+гости → доступные номера с ценой ЗА ПЕРИОД + тарифы (питание,
   refundable/нет — G7 предоплата по тарифу).
2. Номер/тариф → данные гостя (дети-возрасты ✅) → минимум Meldeschein-полей.
3. Оплата по тарифу: 0% (на месте) / частично / 100% (Stripe); Vorkasse если
   настроен.
4. Подтверждение + письмо с суммами (FB-10 ✅) · pre-stay (G2) · online
   check-in (G6) · post-stay отзыв ✅.

**Состояния:** нет номеров на даты → ближайшие альтернативы + календарь
занятости · мультикомнатная (G5) → «+ noch ein Zimmer».

**Вопросы H (дефолты):**
- **H-1**: календарь date-search сразу с занятостью (жирные даты — как на
  детали номера) — дефолт: да.
- **H-2**: карточки номеров видны ДО ввода дат (с «ab €», SEO+обзор) —
  дефолт: да, после поиска заменяются выдачей.

### METZGEREI + BÄCKEREI (гастро-корзина)

**Первый экран butcher:** название + open-status → **«Wochenangebote»**
(акции с ценами, стили UE2, срок действия) + CTA «Vorbestellen» → каталог →
вторым блоком **Partyservice**-карточка с мини-заявкой (jobs).
**Первый экран bakery:** **«Heute frisch»** (подборка дня) + «Jetzt geöffnet»
+ CTA «Vorbestellen»; Wochenangebote вторым блоком.

**Поведение (C&C):** товар (вес/вариант) → корзина → время самовывоза →
контакт → оплата (on-site дефолт гастро / online) → «Abholbereit»-уведомление ✅.
**Partyservice:** мини-форма (дата события · гостей · пожелания) → тред →
Sofort-Angebot LS-3 → оплата по ссылке.

**Состояния:** товар кончился → Warteliste (есть у mode; для гастро — просто
«morgen wieder») · заказ после закрытия → слоты со следующего дня.

**Вопросы MB (дефолты):**
- **MB-1**: Wochenangebote = обычные promotions первым экраном (дефолт) —
  без новой сущности.
- **MB-2**: «Heute frisch» = ручная Collection владельца (дефолт) — не авто.
- **MB-3**: Partyservice — мини-форма 3 поля прямо на главной (дефолт) —
  или карточка-переход на /anfrage/.

## Волна 2

### WERKSTATT

**Первый экран:** **плитки топ-работ с Festpreis** («Inspektion ab 129 €» ·
«Reifenwechsel 39 €» · «TÜV/HU») → клик = сразу слот; рядом «Anderes
Anliegen? → Kostenvoranschlag»; **телефон крупно (click-to-call)** + open-status.

**Поведение booking:** работа → слот → данные авто (Kennzeichen/марка опц.) →
контакт → подтверждение. **Kostenvoranschlag:** форма HSN/TSN ✅ + фото →
тред → смета → принятие онлайн.

**Состояния:** «сломался сегодня» → кнопка «Notfall? Anrufen».

**Вопросы W (дефолты):**
- **W-1**: телефон click-to-call в первом экране — дефолт: да.
- **W-2**: плитки работ = услуги, отмеченные владельцем (дефолт) — не авто-топ.

### HANDWERKER

**Первый экран:** заголовок + **мини-заявка 3 поля** (Was brauchen Sie? ·
фото-кнопка · контакт) + CTA «Unverbindliches Angebot anfordern»; рядом
до/после-референсы + бейджи (Meisterbrief · годы · район) + рейтинг + честное
«⌀ Antwortzeit» (LS-4, порог ≤2ч).

**Поведение:** заявка → «Antwort in ~N Std» → смета `/angebot/<token>/`
(позиции · сумма · срок) → принятие онлайн → (предоплата если настроена) →
работа → счёт (finance ✅).

**Состояния:** вне района (service_area ✅) → честное «wir arbeiten in X».

**Вопросы HW (дефолты):**
- **HW-1**: мини-форма шлёт сразу в jobs без выбора направления (дефолт) —
  направление уточняется в треде.
- **HW-2**: «⌀ Antwortzeit» у формы (честный порог) — дефолт: да.

### EVENTS + TOUREN

**Первый экран events:** **афиша ближайшего события** (крупное фото · дата ·
место · «ab X €» · «N Tickets übrig» · CTA Tickets + countdown) → грид
следующих (RV3 ✅). **Touren:** + date-фильтр и чипы направлений в hero.

**Поведение:** событие → тир (Standard/VIP ✅) → количество → контакт →
оплата → QR-билеты письмом ✅. Распродано → Warteliste ✅.

**Состояния:** прошедшие скрыты ✅ · перенос/отмена события → письмо
участникам ✅.

**Вопросы E (дефолты):**
- **E-1**: countdown на афише главного события — дефолт: да.
- **E-2**: date-фильтр в hero у touren — дефолт: да.

## Волна 3

### RESTAURANT + CAFÉ (решение: два равных CTA)

**Ресторан:** фото-hero + **«Speisekarte» / «Tisch reservieren»** равными
кнопками (+ третья «Bestellen» автоматически при включённых orders) +
open-status + рейтинг. **Кафе:** меню-тизер компакт-прайсом (5–7 позиций,
ST-7c ✅) + «Tisch reservieren» + штампы-тизер.

**Поведение бронь:** персоны+дата → свободное время → контакт →
подтверждение + напоминание ✅. **Заказ:** меню → корзина → время → оплата.

**Вопросы R (дефолты):**
- **R-1**: у кафе бронь равным CTA, как у ресторана — дефолт: да.
- **R-2**: третий CTA «Bestellen» авто-появляется при активных orders —
  дефолт: да.

### MODE + RETAIL + ONLINE-SHOP + GROCERY

**Mode:** hero-витрина новинок (overlay-карточки ✅ полной шириной) + **чипы
размеров** → листинг с прешёлканным размер-фильтром; Sale — пункт меню.
**Retail:** категории-плитки с фото (FB-6 ✅) + хиты; акции вторым.
**Online-Shop:** хиты + **строка доверия «Versand ab X € · Lieferzeit N Tage»
+ бейджи оплаты** сразу под hero (из настроек доставки/E-7).
**Grocery:** категории + Wochenangebote 50/50.

**Поведение cart:** карточка (размер/вариант обязателен ✅) → корзина
(upsell-пресет ST-2 ✅) → доставка/самовывоз → оплата (E-7 микс ✅) →
статус-трекинг + «Abholbereit»/Versand-уведомления ✅.

**Вопросы RT (дефолты):**
- **RT-1**: mode — чипы размеров в hero → листинг с фильтром — дефолт: да.
- **RT-2**: online_shop — строка доверия из настроек доставки автоматически —
  дефолт: да.

### ПРОМО-АРХЕТИП — согласован, вопросов нет

Акции-грид с countdown/остатком первым экраном (авто при primary=promotions)
→ карточка → резерв (имя+телефон, без аккаунта) → код/QR → погашение в
магазине ✅ → отзыв. Anti-oversell ✅.
