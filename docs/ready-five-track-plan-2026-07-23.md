# Трек «5 из готового» — план исполнения (2026-07-23)

Источник: карта направлений `scratchpad/dev-options.html`, секция «С чего бы я
начал — 5 рекомендаций из готового». Владелец: «далее делай по твоему плану».
Идём инкрементально: каждый пункт = свой батч → локальный гейт → CI → FF-merge.

## Пункты (порядок = ROI/эффорт)

### R4 — Степпер + автозаполнение C&C (S) — ПЕРВЫМ (почти бесплатно)
- **Степпер на самой массовой воронке** (розница/пекарня/мясная): `funnels.py`
  уже знает `order`-kind (E5), тег `{% funnel_steps %}` есть — вшить в C&C-воронку
  (корзина → checkout → подтверждение = 1/2/3).
- **Автозаполнение checkout** для залогиненного клиента: `account_customer` уже в
  контексте (имя/email/телефон/адрес) — предзаполнить поля формы заказа.
- Инвариант: гость без аккаунта — поля пустые; смысл шагов реальный (страницы).

### R2 — Первый экран по архетипам (M) — расширение E4
- Механизм `site_defaults.hero_widget` ("" | stays | services) + `_hero_widget.html`
  уже есть (E4; hotel=stays). Гэп — только раскладка на дефолт-шаблонах остальных
  архетипов.
- Раскатить: booking/services-primary (friseur/handwerker/werkstatt/gastro) →
  `hero_widget="services"` (топ-услуги + «Termin buchen» В баннере). Гейт модуля.
- Инвариант: существующие сайты не трогаем (ключ presence-minimal, только дефолт-
  шаблоны/новые применения); `test_looks`/golden целы.

### R3 — Empty-state конверсия (M)
- «Nächster freier Termin» / ближайшие свободные номера — перехват уходящего
  клиента в момент отказа (пустой листинг/поиск без результатов).
- Услуги: следующий свободный слот сервиса → CTA. Номера: ближайший свободный
  диапазон дат → CTA. Витринный слой, без миграций.

### R5 — Аналитика/BI платформы (L) — крупный, последним
- MRR / churn / LTV из billing-данных (SHARED). Платформенная страница
  (public urls, superuser). Данные есть (RevenueEntry/Subscription), витрина —
  считаем заранее (до запуска тарифов).

### R1 — k6 нагрузка на стенде (S) — ВЕРИФИКАЦИЯ (реальный прогон — владелец)
- Скрипт `scripts/load/` готов; «0 oversell на полном стеке» требует боевого
  сервера (Hetzner) — прогон на владельце (Stage 0). Здесь: проверить скрипт
  актуален/корректен + задокументировать команду прогона. Кодового деливери в
  этой среде нет (нет стенда).

## Порядок
R4 → R2 → R3 → R5; R1 = верификация скрипта (owner-run). Детали каждого пункта —
дописываются по разведке перед его батчем (инварианты: classic_ui цел, golden
normalize цел, секции гейтятся модулем, существующие сайты не трогаем).
