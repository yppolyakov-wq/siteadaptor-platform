## 1. Целевая картина (что строим)

**Покупатель:** сказал, что нужно (Finder, 2–4 чипа) → увидел 3 варианта
(средний — «Unser Vorschlag») → при желании — живой человек (чат / видео по
записи / «Jetzt im Laden») → персональное предложение → **оплата в 1 клик** →
care-цикл (напоминания, «Wie war's?», лояльность) → при проблеме — **прямая
линия владельцу** (перехват до плохого отзыва).

**Владелец:** регистрация с подтверждением почты → мастер (язык → товар →
страницы с шаблонами) → **Studio** (Look → Seiten → Blöcke, живая галерея с ЕГО
контентом) → админка **5 хабов** (Заказы-канбан/календарь · Angebot · Marketing
· Integrationen · Einstellungen) с хоумом «сегодня». Всегда доступен откат
«Klassische Ansicht».

## 4. Правила исполнения (обязательные, из уроков сессии)

1. **classic_ui** (Р7): каждый редизайн-инкремент гейтится флагом; легаси-шаблоны/
   маршруты не удаляются. Контекст `classic_ui` уже в processor.
2. **Golden**: новые top-level ключи site_config — только presence-minimal;
   добавление секции в SECTIONS = осознанная регенерация эталонов с записью в
   build-log (процедура сработала на FD-2).
3. **CSS**: после ЛЮБЫХ новых/изменённых шаблонов — `npm ci && npm run build:css`
   (пиненый Tailwind 3.4.17) и коммит `static/css/app.css` (два красных CI —
   урок). Локальный npm без ci тянет v4 — НЕ использовать.
4. **Гейты**: ruff целиком + pytest затронутых модулей `--reuse-db` (+`--create-db`
   при миграциях) + при правке SECTIONS/sellable — полный `apps/tenants`.
   CI — финальный гейт; FF-мерж в main по зелёному; чекпоинт владельцу.
5. **Письма/строки**: немецкие msgid + переводы en/tr/ru/uk в .po (polib-проверка
   на дубли); rate-limit — реюз существующих msgid.
6. **Миграции**: помечать ⚠️ в CLAUDE.md; деплой — владелец.
7. Нетривиальная волна — **план-док до кода** (LS-3 обязателен: выбор модели
   предложения).
