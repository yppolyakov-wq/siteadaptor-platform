# План: правка контента прямо на витрине (inline-content)

Источник: оценка сложности 2026-06-29 (запрос владельца «редактировать контент прямо на витрине»).
Принцип: **текст** данных правим НА МЕСТЕ; цену/фото/структуру — через «✎» к полной форме (безопасно).

## Слои контента
1. **Тексты оформления** (hero/about/cta/заголовки секций) — ✅ есть (`[data-edit]` → `site-inline-edit` → site_config).
2. **Текст данных** (имя/описание товара·услуги·события·номера, i18n) — Фаза 1.
3. **Числа/флаги** (цена, активность) — Фаза 2.
4. **Фото** (фото записи) — Фаза 3.
5. **Структурное** (категория/варианты/комбо) — оставляем в форме кабинета («✎»).

## Архитектура (реюз)
- Паттерн `category_inline_edit` (catalog/views): `@login_required @require_POST`, JSON→БД→204/400,
  `.objects` = AliveManager (tenant-схема активна → скоуп тенанта), `save(update_fields=[...])`.
- Фронт: обработчик iframe в `site_home.html` уже вешает `[data-edit]`/`[data-cat-edit]`; добавляем
  тип «модель-поле» (`[data-edit-model]`), уважающий режим правки SE-9d (`styleEditable(el, editOn)`).
- Безопасность: СТРОГИЙ вайтлист (model,field) — только i18n-текст; цена/связи/фото НЕ через инлайн.

## Подзадачи (каждая → CI-зелёный мерж)
- **P1-1 ✅ бэкенд: `product_inline_edit`** (catalog) — JSON {pk, field, value}, field ∈
  `_PRODUCT_INLINE_FIELDS = {"name","description"}` → пишет `Product.<field>['de']` живого товара.
  Имя пустым не сохраняем; неизвестное поле (напр. `base_price`) → 400. URL `catalog:product-inline-edit`.
  Тесты: имя/описание/пустое-имя/не-вайтлист-поле(цена)/битый-pk. Без миграций.
- **P1-2 фронт:** обработчик `[data-edit-model]` в `site_home.html` (реюз `styleEditable`/`markSaved`,
  POST {model,pk,field,value} в `product-inline-edit`; уважает `editOn`). Разметка имени/описания
  в `templates/storefront/sections/_products.html` (+ карточка детальной товара). Тест рендера карточки.
- **P1-3 «✎» к форме:** на карточке товара (gate владелец/`?preview=1`) маленький «✎» →
  `catalog:product-edit` (для цены/фото/вариантов). Дёшево, закрывает «остальное».
- **P1-4 расширение:** Service/Event/Room (тонкие вью через общий хелпер i18n-инлайн + разметка их карточек).

## Notes/риски
- i18n: правим `['de']` (локаль витрины de); EN — позже.
- Кэш витрины (pagecache SE-5a) бампится на save site_config, НЕ на save модели → публичная
  страница может быть устаревшей до экспирации (как и у `category_inline_edit`). Для владельца превью
  (`?preview=1`) свежее. При расширении — добавить bump на правку данных.
- contenteditable: пустое имя → 400 (фронт откатывает innerText). Числа/структура — НЕ contenteditable.
- gate: `data-edit-model` в шаблоне рендерим всегда; contenteditable вешает только редактор-iframe
  (`site_home`), на публичной витрине JS-инлайна нет. Эндпоинт всё равно `login_required` (на схеме тенанта).
