# DSGVO-ревизия (Hardening H7)

Статус соответствия платформы по состоянию на 2026-06. Код-часть закрыта;
оставшиеся пункты — организационные, на владельце (помечены ⚠).

## Какие персональные данные храним (TENANT-схемы)

| Где | Что | Основание |
|-----|-----|-----------|
| `Customer` | имя, email, телефон, заметка | исполнение договора (бронь), Art. 6 (1) b |
| `Reservation.note` | свободный текст клиента | то же |
| `WaitlistEntry` | имя, email | согласие на одно уведомление о наличии |
| `Notification.recipient` | email адресата (лог отправки) | то же основание, что событие |

На public-схеме клиентских PII нет (только владельцы бизнесов — договорные).
Витрина без трекинг-куки → cookie-баннер не нужен (счётчик просмотров —
анонимный инкремент без идентификаторов).

## Что уже в коде

- **Retention (Art. 5 (1) e):** beat-задача `purge_reservation_pii` обезличивает
  контакты клиентов без активных броней старше `RESERVATION_PII_RETENTION_DAYS`.
- **Отписка:** one-click `/u/<token>/` (UWG §7); Double-Opt-In обязателен до
  любых маркетинговых рассылок (их пока нет).
- **Экспорт по запросу (Art. 15/20):**
  `manage.py dsgvo_customer --schema <tenant> --email <email>` → JSON-выгрузка
  (профиль, брони, лояльность, waitlist, лог уведомлений).
- **Удаление по запросу (Art. 17):** то же с `--delete` — обезличивает Customer
  (агрегатная статистика остаётся без PII), стирает note броней, удаляет
  waitlist-записи, затирает адресата в логе уведомлений. При активных бронях —
  отказ до их завершения (Art. 17 (3): исполнение договора).
- **Право на витрине:** Impressum / Datenschutz / Widerruf — страницы тенанта.

Запуск на проде: `docker compose --env-file .env.prod -f docker-compose.prod.yml
exec web python manage.py dsgvo_customer …`

## Процесс запроса клиента (для владельца)

1. Клиент пишет бизнесу (или нам) с просьбой выдать/удалить данные.
2. Проверить личность (ответ на тот же email достаточен для этих данных).
3. Экспорт: прогнать команду, отправить JSON клиенту (срок — 1 месяц, Art. 12).
4. Удаление: убедиться, что нет активных броней (иначе завершить/отменить),
   прогнать с `--delete`, подтвердить клиенту.

## ⚠ Организационное (на владельце, до боевого запуска)

- AV-Verträge (Art. 28): Hetzner (хостинг), Stripe (платежи), Resend (почта) —
  подписать DPA у каждого процессора.
- Verzeichnis von Verarbeitungstätigkeiten (Art. 30) — реестр обработок.
- Datenschutzerklärung платформы (siteadaptor.de) — сейчас право есть только
  на витринах тенантов.
- Бэкапы: после `--delete` PII живут в старых дампах до их ротации — зафиксировать
  срок хранения бэкапов в Datenschutzerklärung (стандартно ≤ 35 дней).
