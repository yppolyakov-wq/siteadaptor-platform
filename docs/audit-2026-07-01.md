## 4.2 Реализовано правильно (подтверждено)

- **Stripe-вебхук** (`billing/webhooks.py:207-225`): `construct_event` верифицирует подпись на сыром
  теле ДО обработки, невалидная → 400 (fail-closed); дедуп по `event.id` через `cache.add` (TTL 3 сут);
  эскалация статуса подписки — только из подписанного вебхука (self-elevation недостижим).

  явной резолвлёной схемой; signed-token lookups остаются в схеме предъявляющего субдомена (UUID PK →
  cross-schema 404, не cross-read); кэш-ключи неймспейснуты. Утечек между схемами не найдено.
- **Единый rate-limit + honeypot** на прочих публичных формах (бронь/reserve/waitlist/отзыв/Widerruf);
  единственный `csrf_exempt` — Telegram webhook (по назначению).
- **JSON-LD `seo._dumps`** — корректный фикс `</script>`/U+2028/2029/`< > &` для `localbusiness_ld`/
  `offer_ld`/`entity_ld`. Дыра S1/S2 — именно в **не покрытом** этим фиксом коде карты агрегатора.
- **Параметр-тамперинг платежей** исключён (сумма/`days` с сервера, `connect_id` из

- **Секреты:** Fernet (authenticated), нет хардкод-ключей, write-only masked admin, пустые значения
  не шифруются, секреты не логируются.
