> Этот документ — **синтез** девяти детальных отчётов в `docs/market-analysis/`
> (по архетипу + сквозной билдер). Каждый отчёт: текущее покрытие (по коду) +
> бенчмарк рынка DACH + визуальные гэпы + технические гэпы + блок-редактор. Здесь —

с «функций» на три вещи: (1) визуально-UX-подача витрины «как у лидеров рынка»,
(2) «анти-Битрикс» конструктор/онбординг, (3) точечный правовой и платёжный долг
(Widerruf, способы оплаты).** Доки `micro-business-verticals.md` и
`kit-archetype-coverage.md` местами устарели и занижают готовность (см. §1).

## 2. Сквозные темы (повторяются в ≥4 архетипах)

Это самые ценные работы — каждая закрывает несколько архетипов разом.

### T-A. Блок отзывов/рейтинга **на самой витрине** (не только в агрегаторе)
Сейчас `BusinessReview` живёт в агрегаторе; на витрине бизнеса отзывов почти нет
(ретрит — исключение). Доверие — топ-драйвер конверсии у A3/A4/A5/A6/A7/A9.
→ переиспользуемый блок «Bewertungen» на витрине. **Закрывает: A3, A4, A5, A7, A9** (S–M).

### T-B. Trust-Leiste / USP-bar / типизированные значки доверия
Платёжные иконки, «Versand ab X €», «14 Tage Widerruf» (retail); Meisterbetrieb/
Innung/Siegel (Handwerker/Werkstatt); «Familienbetrieb seit…», сертификаты (отель/
ретрит). Сейчас trust — свободный текст. → блок `usp_bar`/`trust_badges` с
типизированными иконками. **Закрывает: A1, A2, A4, A7, A9** (S).

### T-C. Богатая карточка «главного товара» + грид-обложки
Карточки услуг/номеров/событий/блюд тонкие (название+цена) и рендерятся строками.
Рынок — крупные фото-обложки в гриде с дефицитом/рейтингом. → пресет «card/gallery»
для primary-секций + поля «что входит/фото». **Закрывает: A3, A5, A6, A9** (S–M).

### T-D. Профили людей (Team) связаны с функцией
Мастер/преподаватель/Gastgeber — фото+специализация+кнопка «Termin/Buchen bei …».
Сейчас `team` (текст) и `Resource`/`Teacher` не связаны (ретрит частично связал).
→ связь `team` ↔ ресурс/преподаватель. **Закрывает: A3, A5, A6** (M).

### T-E. Галерея с полноэкранным лайтбоксом везде
Лайтбокс уже есть у номеров отеля/ретрита — переиспользовать на товар/услугу/блюдо.
**Закрывает: A1, A4, A5, A7** (S, переиспользование).

### T-F. Структурированные Öffnungszeiten + «открыто сейчас» + JSON-LD
Live-бейдж часов есть, но не везде; структурные часы питают и фильтр агрегатора, и
JSON-LD для Google Map Pack/AI-поиска. **Закрывает: A1, A4, A8, A9** (S–M).

### T-G. Reminder/Drip по согласию (UWG §7) как канал ретеншна
Drip-письма есть у ретрита/stays; для A3 (повторная запись), A9 (TÜV/Service-
Reminder), A1 (брошенная корзина — спорно) — переиспользовать beat-паттерн.
**Закрывает: A3, A9, частично A5/A6** (M).

(правовой риск для уже-живых retail-тенантов). Детали — `a1a2-retail-online-shop.md`.

## 4. «Анти-Битрикс» — стратегический центр (из `z-anti-bitrix-builder.md`)

**Где мы: ~55–60% к северной звезде.** Фундамент правильный и местами обгоняет
конкурентов: секционная JSON-модель (ADI-подобна Wix ADI), live-preview с draft-в-
сессии, инлайн-правка текста (`[data-edit]`), layout-пресеты-гардрейлы, темы без
веб-шрифтов (GDPR), идемпотентный демо-контент, реестр архетипов.

**Три разрыва:** (1) онбординг отдаёт **пустой** сайт; (2) on-canvas правка — только
текст, остальное в формах на **6 экранах**; (3) блоки **нельзя добавлять** (нет «+»,
диспетчер секций — хардкод `if/elif` в `home.html`).

**Единственный самый большой рычаг — онбординг, выдающий наполненный, красивый сайт
за ≤10 шагов** (тема + демо-контент + контакты внутри мастера). Кирпичи уже есть
(`sitetemplates`, `demo.load_demo`, `presets`) — их нужно сшить в линейный поток.

### 4.1 Предлагаемый онбординг ≤10 шагов
Регистрация → тип бизнеса (плитки) → стиль+цвет (превью-карточки шаблонов) → контакты
→ «Beispiel-Inhalte? Ja/Nein» (демо-контент!) → provisioning (фон) → баннер (текст+фото)
→ live-превью (mobile/desktop) → «Fertig 🎉, сайт онлайн» → (опц.) следующие шаги.
Итог: владелец делает 7–8 осмысленных решений и видит **наполненный** сайт.

### 4.2 Блочная модель (поверх той же JSON-схемы, без новых моделей)
- **A. Functional/archetype-блоки** (data-driven, гейт по модулю): `promotions`,
  `products`, `categories`, `events`, `stay_search`, `stay_rooms`, `contact` — есть.
- **B. Content/marketing-блоки**: `hero`, `about`, `process`, `team`, `cta`,
  `testimonials`, `trust`, `faq`, `gallery` — есть.
- **C. НОВОЕ — свободные «простые блоки»** (множественные, с `id`): `text`, `image`,
  `image_text`, `button`, `spacer`/`divider`, `embed` (GDPR 2-Klick). Решает «нельзя
  добавить ещё блок».

### 4.3 Дорожная карта билдера (критпуть)
- **Phase 1 «Сайт не пустой»** (быстрые победы): демо в мастере (S), тема-пикер в
  мастере (S), загрузчик hero-фото файлом (M), линейный `/willkommen/` (M).
- **Phase 2 «Реестр блоков»** (разблокиратор): диспетчер `{% render_block %}` вместо
  if/elif (M) → C-блоки (L) → единый builder-экран 6→1 (M).
- **Phase 3 «On-canvas»**: click-to-edit→попап (M), инсертер «+» (M), Undo/Redo (S),
  drag-on-canvas SortableJS (M).
- **Phase 4 «Vow» (позже)**: импорт из Google Places (L), AI-тексты (L, требует
  решения по провайдеру), независимая mobile-правка (L, низкий приоритет).

**Критпуть:** Phase 1 → 2.1 (реестр) → 3.3/3.1 → 2.2 → 3.2/3.4. Реестр блоков (2.1)
— разблокиратор всего on-canvas. Всё в HTMX/Alpine/Tailwind, без SPA.

## 5. Топ-гэп по каждому архетипу (быстрый справочник)

- **A3 Termin:** блок «Leistungen & Preise» на главной (его нет в `SECTIONS`!) ·

- **A4 Gastro:** food-hero пресет · видимость Kombo/Tagesgericht · диет-иконки ·
  гибкий депозит за стол · виджет брони стола · слоты доставки/Trinkgeld.
- **A5 Hotel:** полноэкранный лайтбокс · **визуальный календарь наличия** (сильнейший
  рычаг) · разбивка цены (PAngV) + рейтинг на странице номера · extras с фото ·
  (отложено: реальные OTA-API — нужны партнёрские ключи владельца).
- **A6 Retreat:** **2-шаговый чекаут** (M20R-4) · agenda-timeline по дням · грид-
  обложки + hero с countdown · QR-билет + check-in сканом · онлайн/Zoom-ретрит · блог.
- **A7 Handwerker:** Referenzen/before-after-галерея · Notdienst-CTA с tel · Meister/

- **A8 Aggregator:** фасетные фильтры+сортировка · богатая карточка бизнеса · карта
  как режим · полный JSON-LD (Map Pack/AI) · self-serve Featured (монетизация без
  комиссии) · (развилка: Stadtgutschein/единая корзина — только как white-label).
- **A9 Werkstatt:** прайс-блок с Festpreis на витрине · отзывы на витрине · структурные
  данные авто (Kennzeichen/HSN-TSN) · Repair-Status трекинг · TÜV/Service-Reminder ·
  (крупно: Reifeneinlagerung).

## 6. Предлагаемый порядок (на согласование)

Принцип: сначала сквозные дешёвые рычаги (бьют по многим архетипам), затем
«анти-Битрикс» фундамент, затем точечная глубина по архетипам.

1. **Спринт A — сквозные витринные блоки (дёшево, бьют по многим):** T-A отзывы на
   витрине + T-B Trust/USP-bar + T-E лайтбокс везде + блок `services` для A3 (его нет).
2. **Спринт B — «анти-Битрикс» Phase 1:** демо+тема внутри мастера, hero-фото файлом,
   линейный онбординг `/willkommen/` ≤10 шагов. **Главный стратегический рычаг.**
3. **Спринт C — правовой долг L1:** Widerrufsbutton + Rechtstexte-Wizard (A1/A2).
4. **Спринт D — «анти-Битрикс» Phase 2:** реестр блоков `{% render_block %}` + C-блоки
   + единый builder-экран. Разблокирует свободную «детскую» сборку.
5. **Спринт E — «анти-Битрикс» Phase 3:** on-canvas (click-to-edit, «+», Undo/Redo).
6. **Далее — глубина по архетипам по приоритету владельца** (календарь наличия отеля,
   2-шаг чекаут ретрита, депозит стола Gastro, авто-данные Werkstatt, Handwerker-кит …).

Параллельно/независимо: демо-обогащения (показать уже готовые невидимые фичи —
комбо/QR-стол/Pass/расходник-смета/сезонные тарифы) — дёшево, по ходу спринтов.

## 7. Идеология (держим курс)
Анти-маркетплейс, own-your-audience, 39 €/мес **без комиссии с оборота**, GDPR-native
(без трекинг-куки, OSM-карты, системные шрифты, Double-Opt-In UWG §7). Монетизация
агрегатора — только self-serve Featured/white-label с фикс-платой, **не** комиссия.
«Ребёнок может собрать сайт» = гардрейлы (пресеты, токены тем), а не свобода сломать.
