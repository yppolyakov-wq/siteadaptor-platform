# UC6 — Editor UX v2: единая лёгкая схема редактирования (план, 2026-07-06)

Запрос владельца (2026-07-06, дословно по смыслу): «куча кнопок редактирования
(Править на сайте / Редактировать дизайн / Панель редактирования) — давай
унифицируем: канва-first, клик по блоку открывает его настройки (шрифт/размер/
цвет/выравнивание для текста, замена/скругление для фото, ширина блока 2/3 и
положение), Просто/Эксперт per-блок, глобальные настройки шаблона отдельно,
блок-шаблоны с демо-данными, адаптивно под все девайсы».

ID — UC6 (углубление волны U-C «универсальный редактор», каталог §3).
Родительский контекст: `uc-plan §11`, Спринт E (on-canvas E.1–E.4), SE-*.

## §1. Инвентарь: что УЖЕ есть (разведка кода 2026-07-06)

**Входы (кнопок реально много — подтверждаю):**
- FAB «✏️ Edit design» на витрине (с T-6.1 — deep-link на текущую страницу);
- «✏️ Edit panel» (`bld-drawer-toggle`) — открыть/закрыть рейл областей;
- «✍️ Edit on site» (`bld-edit-toggle`, sf_edit_on) — режим инлайн-правки
  текстов/фото на канве;
- Simple/Expert — глобальный тумблер (21 `data-expert`-контрол);
- Share preview · Save · Undo/Redo · селектор страниц · Desktop/Tablet/Mobile.

**Канва уже умеет:** клик по секции → попап настроек (`openBlockPopup`:
раскладка list/cols2-4/gallery + per-device колонки, лимит, заголовок,
источник товаров, «View all», visual radius/shadow/bg/padding, скрытие
per-device, ширина contained/full, per-секционный шрифт); «+» инсертер
(6 типов C-блоков + сохранённые блок-шаблоны SE-4a/4c); drag ⠿; инлайн-тексты
(`data-edit`/`data-edit-model`); промо: %-бейдж/цены/дата/фото/стиль скидки
(UE2/UE3); per-page инспектор деталей (UC1-3/UC2-2); история версий.

**C-блоки** (`REPEATABLE_BLOCKS`): text / image / image_text (side) / button /
spacer / promo. У всех: width contained|full, font, hidden_on. Данные — строго
санитайзятся `_clean_cblock_data`.

**Глобальные области рейла:** Theme (акцент/шрифт/hero/типографика SE-3b/стиль
карточек SE-2d), Banner, Sections, Menu (стиль шапки+sticky), Footer, Library,
медиа-области.

## §2. Гэпы против запроса владельца

1. Три кнопки-входа + глобальный Simple/Expert — нет ЕДИНОЙ модели.
2. Текст: НЕТ выравнивания, размера, цвета на уровне блока.
3. Ширина блока: только contained/full — нет 2/3, 1/2 и положения (лево/центр/право).
4. Фото в C-блоках: замена только через URL/библиотеку в панели, скругление
   только у секций-сеток; на канве фото-опы есть лишь у товаров/промо/событий.
5. Библиотека блоков — без превью и без демо-данных при вставке.
6. Рейл сворачивается, но полного «канва-first» режима нет (панель нужна для
   части настроек).

## §3. Целевая схема (v2)

**Два режима вместо трёх кнопок:**
- «✏️ Редактировать» — ОДНА кнопка: включает и клики-по-блокам, и инлайн-тексты
  (объединяет Edit panel + Edit on site; рейл по умолчанию скрыт — канва во
  весь экран);
- «⚙️ Шаблон» — глобальные настройки (тема/типографика/меню/футер/правовое) —
  текущие области рейла, собранные в один режим.

**Клик по блоку → единый попап с вкладками «Просто | Эксперт»** (вместо
глобального тумблера; наполняем постепенно):
- Просто: тип отображения (по контексту блока), заголовок, фото (замена на
  канве + скругление), текст (выравнивание L/C/R, размер S/M/L/XL, цвет из
  палитры темы), ширина блока (full / wide / 2/3 / 1/2) + положение (L/C/R).
- Эксперт: visual (radius/shadow/bg/padding), per-device (колонки/скрытие/
  выравнивание), шрифт, лимиты/источники — всё, что уже есть.

**Хранение:** только site_config JSON (без миграций), normalize со строгими
_clean-канонами; рендер — Tailwind-классы (⚠️ каждый слайс с новыми классами —
`npm run build:css`). Цвет текста — ТОЛЬКО палитра темы (акцент/нейтрали), не
свободный пикер — сохраняем целостность дизайна и «анти-Битрикс» простоту.

**Библиотека блоков:** карточки типов с мини-превью; вставка — с демо-данными
(DE-рыба по типу блока), сохранённые шаблоны владельца — рядом.

## §5. Развилки — РЕШЕНЫ владельцем (2026-07-06, «приступай»)

1. **Цвет текста:** палитра темы ✅ (не свободный пикер).
2. **«Шаблон»:** режим в том же редакторе ✅.
3. **Просто/Эксперт:** вкладки в попапе каждого блока ✅.
4. **Форм-фактор настроек блока (идея владельца):** докнутая ПАНЕЛЬ НАСТРОЕК
   СВЕРХУ канвы («как лента в Word»), с миниатюрами-иконками («как в
   Photoshop»), сворачиваемая/разворачиваемая; на мобильном — тот же набор
   как bottom-sheet снизу (под большой палец). Реализация — UC6-6 (замена
   текущего попапа в левой панели); слайсы UC6-2..4 добавляют КОНТРОЛЫ,
   которые лента потом переиспользует.
