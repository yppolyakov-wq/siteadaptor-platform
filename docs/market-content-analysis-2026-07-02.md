# Рыночный анализ: управление контентом (и рядом) для DACH-SMB — 2026-07-02

> По запросу владельца при старте U-C: «какие ещё функции оптимально добавить для управления
> контентом и не только». База: наш верифицированный корпус `docs/market-analysis/*` +
> свежий веб-срез (июль 2026, источники в конце). Гэпы против нашего кода проверены.
> Дополняет `docs/feature-ideas-2026-07-02.md` (A1–D3) — здесь контент-фокус, идеи CM-*.

## 1. Три рыночных сигнала 2026

**(1) Билдеры стали AI-first.** Jimdo/Wix 2026: AI генерит макет, тексты, ЛОГОТИП и палитру
из анкеты за минуты; Jimdo «Local & SEO» — авто-листинги в Google/каталоги + AI-SEO.
Порог «красивый сайт» упал — конкурировать надо скоростью НАПОЛНЕНИЯ и вертикальной глубиной
(бронь/анти-оверселл/право DACH — наша сила, у билдеров этого нет).

**(2) Google Business Profile — центр локальной видимости, «свежесть» = ранкинг-сигнал.**
Профиль без постов/фото 30+ дней теряет видимость; выдачу всё чаще читают AI-ассистенты
(AI Overview/Gemini/ChatGPT) прямо из GBP. Рыночная норма 2026: планирование постов наперёд,
bulk-постинг, авто-запрос отзывов через 24ч после визита, AI-ответы на отзывы (3× скорость
реакции). → Контент-автопилот вокруг GBP — самая горячая зона.

**(3) Вертикальный SaaS (Fresha/Treatwell/Shore/resmio) продаёт «управление клиентами», но
маркетплейс владеет клиентом.** Fresha бесплатна, зарабатывает комиссией с новых клиентов
маркетплейса; Treatwell — гибрид система+маркетплейс. Боль, которую они создают: «клиентская
база принадлежит порталу, не студии». resmio (5000+ ресторанов DACH): гостевая база, приватный
фидбек, цифровое меню, телефонный ассистент. → Наше позиционирование «клиент принадлежит
бизнесу» + CRM-lite — прямой ответ на эту боль.

## 2. Что у нас УЖЕ есть (и редкость в этом ценовом слое)

Органик-постинг в GBP/FB/IG/Telegram/Pinterest с in-app OAuth (✓ уникально за ~39 €); блог
(RT4); newsletter c Double-Opt-In; post-visit запрос отзыва по всем kind (закрыт 2026-07-02);
generic-отзывы с верификацией покупателя; Telegram-бот/Mini App; QR-стол/меню; клиентская
`Customer` с тегами; демо-контент 9 китов; SEO JSON-LD на всех деталях.

## 3. Гэпы и предложения (CM-1…CM-9; verified против кода)

### Группа I — Контент-хаб (ядро «управления контентом»)

| # | Что | Гэп (verified) | Ценность | Разм. |
|---|---|---|---|:--:|
| **CM-1** | **Блог/новости — first-class для ВСЕХ архетипов** (перенести/генерализовать из events; «Neuigkeiten» пекарни/салона без модуля событий) | `BlogPost` живёт в `apps/events`, сидер гейтится `is_module_active("events")` | свежий контент = локальное SEO; проще некуда для владельца | S/M |
| **CM-2** | **Контент-календарь + отложенная публикация** — планировать посты (блог + соцсети + GBP) наперёд, batch-режим | `Publication` без `scheduled_at`, только queued; постится немедленно и только промо | «freshness-автопилот»: рыночная норма 2026, ранкинг-сигнал | M |
| **CM-3** | **Авто-посты из событий платформы** — новая акция/событие/товар/пост блога → готовый черновик поста в подключённые каналы (1 клик или авто-правило) | `Publication.promotion` FK — только акции; товары/события/блог не постятся | контент делается «сам», сильный дифференциатор vs Jimdo/Fresha | M |
| **CM-4** | **Медиа-библиотека тенанта** — единое хранилище фото с переиспользованием, alt-текстами (a11y/SEO), позже AI-теги | центральной библиотеки нет (фото прикреплены к сущностям) | порядок в контенте; фундамент пресетов U-C и AI-фич | M/L |
| **CM-5** | **AI-контент-пакет** (расширение идеи A2): тексты hero/about/FAQ + описания товаров/услуг из фото + черновики постов (CM-3) + EN-переводы в L3-оверлей | нет AI-генерации (подтверждено и рыночным доком z-anti-bitrix) | паритет с Jimdo/Wix-2026 + вертикальная глубина; ядро Pro-тарифа | M/L |

| # | Что | Гэп | Ценность | Разм. |
|---|---|---|---|:--:|

### Сознательно НЕ берём (сейчас)
- **Телефонный AI-ассистент** (resmio) — тяжёлая интеграция телефонии, партнёрская история.
- **NAP-синк в каталоги/авто-листинги** (Jimdo Local) — внешние агрегаторы, external-backlog.
- **POS-интеграция** (Shore) — вне SMB-тира, уже отмечено в Tier 4 капстоуна.

## 4. Как это ложится в план

- **CM-1 + CM-2** — естественное продолжение U-C (контент = блоки; календарь = кабинет), можно
  волной «Контент-хаб» сразу после критпути U-C или параллельной веткой (файлы не пересекаются).
- **CM-3** — поверх CM-2 + существующего publishing (генерализация `Publication` на не-промо).
- **CM-5 (AI)** — премиум-модуль Pro-тарифа (D1 из feature-ideas); первым слайсом — тексты из
  анкеты (без внешних фото-моделей), ключ владельца — external-backlog (LLM-провайдер).
- **CM-6/CM-8** — кандидаты «между волнами» (M, самостоятельная ценность).
- Порядок предлагаю решать ПОСЛЕ старта U-C: критпуть редактора первичен, контент-хаб — его
  главный «потребитель» (пресеты/блоки без контента пусты).
