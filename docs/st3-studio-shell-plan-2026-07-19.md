# ST-3 «Studio-оболочка» — план (2026-07-19)

Этап B3 ТЗ `next-gen-master-tz-2026-07-19.md §3` (концепт `studio-concept-2026-07-18.md`,
имя «Studio» — Р6). Разведка (фоновый Explore 2026-07-19, карта файл:строка в отчёте)
показала: **полноэкранная оболочка уже построена** — `#bld-root fixed inset-0` (SE-6),
правый инспектор со свёрткой/ресайзом, тонкий тулбар (Undo/Redo/девайсы/Save UC6-10),
двойная буферизация канвы (`swapPreview`/`push`/`instrumentFrame`), Look-переключение
без перезагрузки (ST-1b), история версий. **ST-3 = переупаковка существующего
`site_home.html`, НЕ новый редактор.**

## 1. Решения

- **Рейка уровней Look/Seiten/Blöcke/Medien** — новый узкий вертикальный столбец
  слева в `#bld-root` (левый рейл W1 был удалён — возвращаем ДРУГОЙ, уровневый):
  кнопки = тонкие обёртки над существующими вызовами, `showArea()`/`data-area`
  НЕ трогаются: **Look** → `showArea('theme')` + скролл к Look-fieldset ·
  **Seiten** → открыть page-ленту (п.2) · **Blöcke** → существующий инсертер/
  `showArea('library')` · **Medien** → `showArea('gallery-media')` + ссылка на
  `media-library`. НОВЫЕ id (`#st-rail`, `.st-level`) — существующие id/классы
  не переименовываются (замки test_home_builder жёстко их фиксируют).
- **Page-лента снизу** — горизонтальный ряд страниц из `preview_pages`
  (уже в контексте вью) + текущая подсвечена по `previewPath`; клик =
  `reloadBuilderPage(path)` (существующий). Лента читает/пишет ТОТ ЖЕ
  `previewPath`/`#bld-page-path` (иначе Save уведёт канву не туда — риск
  разведки). Варианты раскладки страницы остаются в области library
  (page-лента v1 = навигация, не пикер вариантов).
- **Кросс-фейд свопа канвы** — ~5 строк в `swapPreview`: буфер уже создаётся
  `visibility:hidden` поверх старого кадра → на флипе opacity-transition
  (150–200мс), `removeChild(old)` по `transitionend` (+ position:absolute
  верхнему до конца перехода); opacity ортогональна `transform: scale()`
  applyDevice — конфликта нет; страховка `hardReloadPreview` цела.
- **classic_ui — вариант A (гейты в одном шаблоне)**: рейка и page-лента —
  чистые ДОБАВКИ к существующей структуре (не перестройка), поэтому обёртки
  `{% if not classic_ui %}` вокруг двух новых узлов дешевле и безопаснее
  отдельного шаблона; при classic билдер выглядит ровно как сейчас.
  (Вариант B — отдельный site_studio.html — оставлен на случай, если ST-2
  потребует структурного расхождения.)
- **Брендинг «Studio»**: заголовок тулбара/вход. Хаб-плитка «Website» (ST-4a)
  уже ведёт на site-home — подпись становится честной. FAB «Edit design» не
  трогаем (T-6 гейты выстраданы).
- **Мобайл**: рейка уровней на <sm сворачивается в нижний таб-ряд НАД
  page-лентой не влезает — v1: на мобиле рейка скрыта (уровни доступны
  прежними кнопками тулбара ☰/🧱), page-лента скрыта; Studio-хром = десктоп
  прежде всего (канва-мобайл остаётся через девайс-переключатель). Отмечено
  как ограничение v1.
- **applyDevice**: добавление левой рейки/нижней ленты меняет геометрию
  `#bld-preview-pane` — синхронно обновить CSS-отступы (left/bottom) и
  положиться на существующий ResizeObserver для масштаба.

## 2. Инкременты

1. **ST-3a Рейка + брендинг**: `#st-rail` (4 уровня, SVG-иконки из спрайта
   ST-4a + подписи), заголовок «Studio», гейт classic_ui, отступ канвы слева,
   desktop-only. Замки: рейка есть в новом виде/нет в classic; существующие
   id целы; showArea-вызовы работают (клик Look открывает theme).
2. **ST-3b Page-лента**: партиал ленты снизу (preview_pages, активная
   страница), клик → reloadBuilderPage; гейт classic_ui; отступ канвы снизу.
   Замок: лента синхронна с `#bld-page-path` после навигации канвы.
3. **ST-3c Кросс-фейд**: opacity-transition в swapPreview + стенд-проверка
   (Playwright: своп без белой вспышки, скролл перенесён, Undo жив).
4. **Финал**: i18n, CSS, build-log/CLAUDE.md/ТЗ-статус, CI → merge.
