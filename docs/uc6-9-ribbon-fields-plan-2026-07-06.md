# UC6-9 — оптимизация пространства ленты + настройки-«поля» (план, 2026-07-06)

Фидбэк владельца (скриншот со стрелками): «оптимизация пространства. Перенеси
важные пункты в верхнюю линию. Далее пункты в таком виде — заголовок, под ним
параметр, далее блоками в строчку остальные параметры так же. Раздели их
визуально, чтоб отличались.»

## Что меняем

### Часть A — верхняя линия (экономия строки)
Сейчас над канвой ОТДЕЛЬНАЯ тонкая полоса: Undo/Redo · статус · «Click text to
edit». Переносим Undo/Redo + статус + подсказку правки в ВЕРХНИЙ тулбар (та же
строка, что Dashboard/девайсы/Menu/Blocks/Template/Save), полосу над канвой
убираем → канва выше, на одну строку меньше. Все id (`home-undo`/`home-redo`/
`home-prev-status`/`bld-edit-hint`) сохраняем → JS не трогаем.

### Часть B — настройки ленты как «поля»
Каждый параметр = маленький визуальный БЛОК: мелкий ЗАГОЛОВОК сверху, контрол
снизу; блоки идут в строчку (flex-wrap) и визуально разделены (рамка + лёгкий
фон + скругление). Реализация:
- Ribbon-open CSS: `> label` → `flex-direction: column` (заголовок над контролом)
  + рамка/фон/паддинг/радиус; заголовок — uppercase 10px, контрол — 12px.
- Секции: чекбокс-лейблы (Round corners/Shadow) переставить на текст-первым, чтобы
  заголовок был сверху (клик по label всё равно тогглит чекбокс).
- C-блоки (`_cb_row.html`): голые селекты (align/size/color, width/pos) обернуть в
  `<label>Заголовок <select></label>` — единообразные поля и в форме, и в ленте.
- Разделитель `border-left` заменяем на рамку блока.

## Замки / проверка
- Рендер-замок: `_cb_row.html` содержит заголовки полей (align/size/color/width/pos).
- Визуальная проверка на стенде (скриншот): секция hero + C-блок в ленте →
  поля-блоки, заголовок над контролом; Undo/Redo/статус в верхней строке; полосы
  над канвой нет. Показать владельцу → итерация.
- БЕЗ бэкенда/миграций; collect()/Save не трогаем (имена контролов те же).
