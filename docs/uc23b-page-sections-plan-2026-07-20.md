# UC2-3(b) «Секции на всех страницах» — план (2026-07-20, «да» владельца)

Вход: `uc2-3-page-scope-plan-2026-07-02.md §3` (владелец дал ДА 2026-07-19).
Разведка (Explore, 2026-07-20): «C-блоки вне home» из пакета УЖЕ закрыты волной
UC6-7 (`page_blocks` на 11 хостах + инсертер/drag/live). Честный остаток —
РЕЕСТРОВЫЕ секции-справочники (faq/team/gallery/testimonials) на не-home
страницах. Ключевой факт: ДАННЫЕ этих секций глобальны (site.faq/team/…,
normalize плоские ключи) — «секция на странице» = показ ОБЩЕГО справочника.

## §1 Принятая трактовка (вариант A разведки)

**Ссылочные секционные блоки**: новые типы C-блоков `faq_ref`/`team_ref`/
`gallery_ref`/`testimonials_ref` — рендерят готовый секционный партиал с
ГЛОБАЛЬНЫМ справочником (тот же контент, что на главной; правится в одном
месте — честная семантика «показать блок FAQ и здесь»). Для уникального
per-page контента уже есть text/image/image_text-блоки. Полноценные per-page
ДАННЫЕ секций (вариант B: новый ключ page_sections + per-page редакторы) —
ОТДЕЛЬНОЕ решение владельца с golden-бюджетом; в отчёте предложим как опцию.

- Ref-типы разрешены ТОЛЬКО в `page_blocks` (реестр `PAGE_REF_BLOCKS`);
  в home-`sections` НЕ допускаются (там живут настоящие секции — дубль
  запутал бы; normalize_sections не расширяется → golden целы).
- `page_blocks` presence-minimal, эталоны его не содержат → golden-риск ~0.
- legal-страницы — по-прежнему без блоков (DACH, PAGE_BLOCK_HOSTS без legal).

## §2 Реализация

1. `siteconfig.py`: `PAGE_REF_BLOCKS = ("faq_ref","team_ref","gallery_ref",
   "testimonials_ref")`; `normalize_page_blocks` принимает
   `REPEATABLE_BLOCKS + PAGE_REF_BLOCKS`; `_clean_cblock_data` для ref-типов →
   {} (v1 без осей; width/pos/visual — общие оси C-блока работают).
2. Рендер: `siteui.render_block`/CBLOCK-диспетчер += ref-типы → обёртки,
   включающие существующие партиалы секций (`_faq/_team/_gallery/
   _testimonials.html`); тег `page_blocks` уже резолвит site-конфиг — прокинуть
   `site` в контекст партиалов (проверить текущую передачу).
3. Инсертер страниц: ref-типы в библиотеке блоков ТОЛЬКО при `curPbHost`
   (страница), не на главной; server-side гейт `add_block` расширить для
   page_key-веток. Лейблы DE («FAQ anzeigen», «Team anzeigen», …).
4. Пустой справочник (site.faq пуст) → блок рендерит пусто (партиалы уже
   гейтятся наличием данных) — честно, без заглушек на публике; в превью —
   существующий пунктирный якорь страниц.

## §3 Замки

- normalize: ref-блок в page_blocks переживает normalize; ref-блок в home
  sections ОТБРАСЫВАЕТСЯ; golden байт-в-байт (page_blocks нет в эталонах).
- Рендер: вставленный faq_ref на /produkte/ показывает site.faq; пустой
  справочник → пусто.
- Билдер: add_block(faq_ref, page_key=catalog) сохраняет; на главной тип
  недоступен (гейт).
- test_home_content_blocks_details_is_home_scoped и паритеты страниц целы.

## §4 Инкременты

1. Реестр+normalize+рендер+тесты. 2. Инсертер+гейты+тесты+CSS при новых
   классах. 3. Докблок + отчёт владельцу (объяснить трактовку A и опцию B).
