# On-canvas редактор на витрине — план и подзадачи

> Запрос владельца (2026-06-27): редактировать сайт **прямо на витрине** — кнопкой
> «Изменить дизайн», тулбары/инспектор блоков на канве, правка любой страницы,
> переиспользуемые блоки и микрошаблоны, **левая** панель «как в фотокниге» (пресеты+
> шрифты), переключение устройств и пер-девайс правка, публикация с предпросмотром.
> Выбранный объём старта: **Стадия 1+2** (редактор на витрине + инспектор блока).

## Принципы
- **Только владелец** (Django-аутентификация на витрине = владелец; клиенты — отдельный
  `apps.account`). Публичная витрина для посетителей не меняется и не нагружается.
- **Черновик → публикация.** Правки копятся в черновике (Redis-сессия, эфемерно, TTL),
  витрина рендерит его по `?preview=1`; «Опубликовать» = одна запись JSON в `site_config`.
  «Отмена» сбрасывает черновик.
- **Переиспользуем существующее**: home_builder + live-iframe, `site_preview_draft`,
  обёртки `data-sf-section`, реестр секций/C-блоков, пресеты раскладки, `sitetemplates`,
  пары шрифтов, переключатель устройств, per-page layouts. Бэкенд по максимуму не трогаем.
- **Scope настроек:** глобальные (шрифт/акцент/шапка/подвал/стиль карточек) → «весь сайт»;
  по-страничные/по-блочные (раскладка/число/тексты) → «эта страница / выбранные».

## Нагрузка (оценка)
Посетители — 0 доп. нагрузки (тот же статичный JSON-рендер). Редактирование редкое,
«всплесками»: черновик в Redis, пере-рендер превью с debounce, публикация — 1 запись.
Опц. кэш витрины (`pagecache`) со сбросом при публикации. Масштабируется на тысячи тенантов.

## Режимы инспектора: «Обычный» и «Эксперт» (progressive disclosure)
Как в редакторах фотокниг — два уровня в левой панели, переключатель вверху:
- **Обычный** — несколько дружелюбных контролов на блок: *скруглить углы* (radius фото/
  баннера/карточек), число/колонки крупными кнопками, скрыть, переместить, сменить
  микрошаблон. Цель — «ребёнок соберёт».
- **Эксперт** — раскрывает всё: источник данных, точные отступы/радиусы/тени,
  типографика, per-devais оверрайды, scope «весь сайт/страница». Выбор режима
  запоминается (localStorage).
Новые **визуальные параметры блока** (radius/тень/фон) — это и есть «простые» контролы;
хранятся в `site_config` рядом с layout, применяются в рендере секций/карточек.

## Подзадачи

### SE-1 — Вход с витрины + инспектор блока на канве (Стадия 1+2)
- **SE-1a** ✅ Кнопка «✏️ Edit design» на витрине (только владельцу, скрыта в `?preview=1`)
  → открывает on-canvas редактор. (готово, тест `test_owner_edit_button`.)
- **SE-1b** Клик по секции на канве (iframe) → подсветка + **инспектор слева** для этого
  блока (панель в home_builder, наполняется по выбранному `data-sf-section`).
- **SE-1c** ✅ Инспектор: базовые параметры секции-сетки — **колонки** (пресет раскладки),
  **число карточек** (лимит), **источник** (избранное/новое, expert), **скрыть**, **переместить
  ↑▼**. Действия правят инпуты формы → пере-сериализация черновика → превью. Save публикует.
  Кнопки ↑▼ — value-based (правят `order_`-инпуты, видны в обоих режимах, в отличие от
  drag-ручки/числа порядка — они expert); список и кнопки-края синхронизируются.
- **SE-1d** Инлайн-правка текста выбранного блока в инспекторе (заголовок секции и т.п.),
  синхрон с инлайн-правкой на канве.
- **SE-1f** ✅ Переключатель **«Обычный / Эксперт»** в инспекторе (progressive disclosure):
  обычный — несколько простых контролов; эксперт — все. Запоминается (localStorage,
  ключ `sf_editor_mode`; экспертные контролы помечены `data-expert`, скрыты CSS в basic).
- **SE-1e** Тесты + доки (структура инспектора, маппинг действий на поля формы).

### SE-2 — Тот же редактор на остальных страницах
- **SE-2a** Каталог `/sortiment/` и страница **категории** — канва + инспектор (раскладка,
  число, подкатегории-блоки), per-page настройки.
- **SE-2b** Лендинги архетипов (события/номера/услуги) и **детальная** — секции детальной
  (реестр уже есть для событий) в инспекторе.
- **SE-2c** «Добавить категорию → сохранить → зайти и править» — связка каталога с канвой.
- **SE-2d** Scope-переключатель «эта страница / весь сайт / выбранные» в инспекторе.
  **РЕШЕНИЕ ВЛАДЕЛЬЦА (2026-06-28):** делаем **Вариант A** — настоящая мастер-настройка
  «на весь сайт»: вводим глобальный дефолт (напр. `site_defaults` со стилем карточек/
  раскладкой), который наследуют все сетки при отсутствии пер-страничного override; в
  инспекторе scope-переключатель выбирает, КУДА писать (глобально vs текущая страница).
  «Выбранные»: в SE-2d — только «эта страница / весь сайт» + лёгкая кнопка «применить
  раскладку ко всем лендингам (каталог+события+номера) разом»; полный мультивыбор
  страниц — отдельной задачей позже. Вариант A = доработка рендера (наследование),
  миграций нет (JSON), но риск регрессий → ИДЁМ ПЛАН-ДОКОМ ДО КОДА, мелкими шагами,
  с регрессионными тестами на legacy-конфиги. (План-разведка SE-2d — 2026-06-28.)

#### Дизайн SE-2 (по разведке 2026-06-27 — конкретика и разбивка)
**Что уже есть (переиспользуем, НЕ переписываем):**
- Чтение черновика **page-agnostic**: контекст-процессор `apps/core/context.py` накладывает
  `session["site_preview_draft"]` на ЛЮБОЙ странице при `?preview=1` (через `normalize`). Значит
  per-page раскладки (`catalog_layout`/`events_index_layout`/`stay_index_layout`/`detail_related_layout`)
  из черновика УЖЕ применятся на `/sortiment/?preview=1` и т.п.
- Per-page раскладки уже редактируются на вкладке «Pages» (`pages_view`, `site_pages.html`) и
  пишутся в `site_config`. `grid_class_string`/`normalize_layout` — общие.
- Инлайн-правка текста `site-inline-edit` пишет в БД page-agnostic (нужен лишь `data-edit` в шаблоне).

**Чего НЕ хватает:**
1. Превью-iframe редактора жёстко грузит `/?preview=1` (только главная) — `push()` в `site_home.html`.
2. На шаблонах `products.html`/`event_index.html`/`stay_index.html`/`service_index.html` и
   детальных НЕТ обёрток `data-sf-section` и `data-edit`.
3. Нет on-canvas инспектора per-page раскладки (он на отдельной вкладке Pages).

**Разбивка на маленькие CI-зелёные инкременты:**
- **SE-2a-1** Переключатель страниц в редакторе (Home/Каталог/События/Номера/Услуги — только
  имеющиеся у тенанта). Меняет URL iframe-превью (`/`, `/sortiment/`, `/veranstaltung/`,
  `/unterkunft/`, `/termin/`) + `?preview=1`. `push()` параметризуется текущей страницей
  (грузит выбранный URL, а не хардкод `/`). Низкий риск (надстройка над iframe).
- **SE-2a-2** На `products.html` (каталог) — `data-sf-section="catalog"` на грид-секции +
  `data-edit` на заголовок; клик по гриду → инспектор с раскладкой каталога (тот же набор
  пресетов), запись в черновик `catalog_layout` → live-preview. Инспектор per-page рисуем
  отдельной мини-формой (не перенос home-block-строк).
- **SE-2a-3** Страница категории — то же (раскладка наследует `catalog_layout` или свой ключ).
- **SE-2b-1** Лендинги событий/номеров/услуг — `data-sf-section`+`data-edit`+инспектор раскладки
  (`events_index_layout`/`stay_index_layout`; услуги — добавить ключ при необходимости).
- **SE-2b-2** Детальная события — инспектор порядка/видимости тематических секций (реестр
  `event_detail` уже есть; сейчас правится на Pages) на канве. Детальные товара/номера —
  пока только инлайн-тексты (полноценный реестр секций детальной — отложен, см. SE-3 «реестр»).
- **SE-2c** Связка «добавить категорию → править на канве».
- **SE-2d** Scope-переключатель «эта страница / весь сайт».

**Принцип:** черновик остаётся ОДНИМ `site_config`-объектом (per-page ключи внутри него) —
не плодим per-page черновики в сессии; это сохраняет page-agnostic чтение и простоту публикации.

### SE-3 — Микрошаблоны, шрифты, устройства
- **SE-3a** **Библиотека микрошаблонов** на тип блока (4–8 готовых обликов: сетка/слайдер/
  список, плотность, тень/рамка) — клик применяет пресет к блоку.
- **SE-3b** **Типографика в инспекторе**: пары шрифтов (системные стеки, GDPR), размер,
  начертание — глобально и/или на блок.
- **SE-3d** 🟡 (radius/тень ✅; фон/отступы — далее) **Визуальные параметры блока**:
  скругление углов (radius), тень, фон, отступы — на карточки; рендер из `site_config`
  (`visual={radius,shadow}` per-секция). **Обычный** — «Round corners» вкл/выкл (→16px);
  **Эксперт** — точный radius (slider 0–24) + тень. Рендер: класс `.sf-card` на карточках +
  CSS `[style*="--sf-r"] .sf-card{… !important}`; обёртка секции несёт inline `--sf-r`/
  `--sf-sh` лишь когда значение задано → ненастроенные витрины без регрессии. Осталось:
  фон/отступы + расширение на фото/баннер.
- **SE-3c** **Устройства**: переключатель Desktop/Tablet/Mobile в редакторе (есть) +
  **пер-девайс оверрайды** ключевых параметров (колонки/видимость/порядок; основа —
  существующий `…_layout.mobile`). Правка прямо в мобильном виде.

## Порядок и риски
Идём SE-1 → SE-5; каждая под-подзадача — отдельный CI-зелёный мерж. SE-1/2 — низкий риск
(надстройка над формой/черновиком). SE-4c (drag на живой канве) и SE-5b (версии) — выше
риск/объём, делаем позже. Крупные доработки — план-доком до кода (этот файл — source of truth).
