# On-canvas редактор на витрине — план и подзадачи

> Запрос владельца (2026-06-27): редактировать сайт **прямо на витрине** — кнопкой
> «Изменить дизайн», тулбары/инспектор блоков на канве, правка любой страницы,
> переиспользуемые блоки и микрошаблоны, правая панель «как в фотокниге» (пресеты+
> шрифты), переключение устройств и пер-девайс правка, публикация с предпросмотром.
> Выбранный объём старта: **Стадия 1+2** (редактор на витрине + инспектор блока).

## Принципы
- **Только владелец** (Django-аутентификация на витрине = владелец; клиенты — отдельный
  `apps.account`). Публичная витрина для посетителей не меняется и не нагружается.
- **Черновик → публикация.** Правки копятся в черновике (Redis-сессия, эфемерно, TTL),
  витрина рендерит его по `?preview=1`; «Опубликовать» = одна запись JSON в `site_config`.
  «Отмена» сбрасывает черновик.
- **Переиспользуем существующее**: home_builder + live-iframe, `site_preview_draft`,
  обёртки `data-sf-section`, реестр секций/C-блоков, пресеты раскладки, `sitetemplates`,
  пары шрифтов, переключатель устройств, per-page layouts. Бэкенд по максимуму не трогаем.
- **Scope настроек:** глобальные (шрифт/акцент/шапка/подвал/стиль карточек) → «весь сайт»;
  по-страничные/по-блочные (раскладка/число/тексты) → «эта страница / выбранные».

## Нагрузка (оценка)
Посетители — 0 доп. нагрузки (тот же статичный JSON-рендер). Редактирование редкое,
«всплесками»: черновик в Redis, пере-рендер превью с debounce, публикация — 1 запись.
Опц. кэш витрины (`pagecache`) со сбросом при публикации. Масштабируется на тысячи тенантов.

## Подзадачи

### SE-1 — Вход с витрины + инспектор блока на канве (Стадия 1+2)
- **SE-1a** ✅ Кнопка «✏️ Edit design» на витрине (только владельцу, скрыта в `?preview=1`)
  → открывает on-canvas редактор. (готово, тест `test_owner_edit_button`.)
- **SE-1b** Клик по секции на канве (iframe) → подсветка + **инспектор справа** для этого
  блока (панель в home_builder, наполняется по выбранному `data-sf-section`).
- **SE-1c** Инспектор: базовые параметры секции-сетки — **колонки** (пресет раскладки),
  **число карточек** (лимит), **источник** (избранное/новое), **скрыть**, **переместить ↑↓**.
  Действия правят инпуты формы → пере-сериализация черновика → превью. Save публикует.
- **SE-1d** Инлайн-правка текста выбранного блока в инспекторе (заголовок секции и т.п.),
  синхрон с инлайн-правкой на канве.
- **SE-1e** Тесты + доки (структура инспектора, маппинг действий на поля формы).

### SE-2 — Тот же редактор на остальных страницах
- **SE-2a** Каталог `/sortiment/` и страница **категории** — канва + инспектор (раскладка,
  число, подкатегории-блоки), per-page настройки.
- **SE-2b** Лендинги архетипов (события/номера/услуги) и **детальная** — секции детальной
  (реестр уже есть для событий) в инспекторе.
- **SE-2c** «Добавить категорию → сохранить → зайти и править» — связка каталога с канвой.
- **SE-2d** Scope-переключатель «эта страница / весь сайт / выбранные» в инспекторе.

### SE-3 — Микрошаблоны, шрифты, устройства
- **SE-3a** **Библиотека микрошаблонов** на тип блока (4–8 готовых обликов: сетка/слайдер/
  список, плотность, тень/рамка) — клик применяет пресет к блоку.
- **SE-3b** **Типографика в инспекторе**: пары шрифтов (системные стеки, GDPR), размер,
  начертание — глобально и/или на блок.
- **SE-3c** **Устройства**: переключатель Desktop/Tablet/Mobile в редакторе (есть) +
  **пер-девайс оверрайды** ключевых параметров (колонки/видимость/порядок; основа —
  существующий `…_layout.mobile`). Правка прямо в мобильном виде.

### SE-4 — Переиспользуемые блоки и шаблоны страниц
- **SE-4a** «Сохранить блок как шаблон» + «Вставить шаблон» (блок-пресет, многоразовый).
- **SE-4b** Шаблоны страниц (набор секций целиком; поверх `sitetemplates`).
- **SE-4c** Перетаскивание блока на канве + инсертер «+» между секциями (есть основа E.3/E.4).

### SE-5 — Производительность и публикация (опц.)
- **SE-5a** Кэш отрендеренной витрины (`pagecache`) + **сброс при публикации**.
- **SE-5b** Автосейв черновика + «История версий» site_config (откат публикации).

## Порядок и риски
Идём SE-1 → SE-5; каждая под-подзадача — отдельный CI-зелёный мерж. SE-1/2 — низкий риск
(надстройка над формой/черновиком). SE-4c (drag на живой канве) и SE-5b (версии) — выше
риск/объём, делаем позже. Крупные доработки — план-доком до кода (этот файл — source of truth).
