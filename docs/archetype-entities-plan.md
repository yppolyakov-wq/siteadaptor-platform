# Архетипы как сущности + мультиархетип + гейтинг настроек — план H0–H4

Согласовано с владельцем 2026-06-29. Источник правды по этому этапу. Строим **поверх
M20U** (единая главная/каталог/детальная, per-page раскладки, реестры
`archetypes.primary_item`/`purchase_mode`) — НЕ с нуля.

## Типы сайтов (3)
1. **Одно-архетипный базовый** — дефолт для самостоятельных пользователей (один тип услуги).
2. **Сборный мультиархетипный** — несколько архетипов в одном сайте (как `pranasy`);
   включаем **МЫ** (с нашей помощью), не сам пользователь.
3. **Агрегаторы** — платформенные, управляем мы, **два вида**:
   - по архетипу (напр. `hotels.siteadaptor.de`),
   - **по городам**.
   Настройки пока минимальны → отдельный трек позже (но в реестре типов заложить).

## Каркас: архетип = 3 сущности-страницы
- **Главная** (лендинг) · **Категория/список** (витрина items) · **Товар/услуга** (деталь).
- Маппинг архетип → (список, деталь):
  - catalog → категории / товар
  - booking → услуги / услуга
  - stays → номера / номер
  - events → события / событие
  - jobs → типы работ / заявка-смета

## Решения (Q1–Q5)
- **Q1** Формализуем реестр НА архетип: `home-секции` + `страница-список` + `страница-деталь`
  + набор применимых секций/настроек/полей.
- **Q2** Настройки секции = **визуал** (width, шрифт, layout, отступы/фон) +
  **контент** (заголовок, описание, слайдер вкл/фото, источник items, лимит, «View all»).
- **Q3** Меню в мультиархетипе — **авто** по активным архетипам (пункт на архетип → его список),
  но **редактируемо** владельцем.
- **Q4** Страница «Категория» = сущность «список»: описание + слайдер + грид +
  **подкатегории + фильтры**.
- **Q5** Главная мультиархетипа — порядок блоков архетипов **авто по реестру** +
  ручная перекомпоновка в билдере.

## Гейтинг настроек/полей (анти-Битрикс)
- Все настройки/поля/секции/блоки **сопоставлены архетипу**; нерелевантное — **скрыто**.
- Одно-архетип: 3 сущности + секции/настройки только своего архетипа; ОДНА точка
  «включить больше — с нашей помощью» (не россыпь заблокированных контролов).
- Мульти: объединение архетипов + контролы композиции (меню-категории, агрегация главной).
- Доп. архетипы включаем **только МЫ** (админ), не пользователь в обычном кабинете.

## План (порядок, согласован)
- **H1.1 — Баг** «клик по ссылке в редакторе → ошибка». ✅ **СДЕЛАНО.** Корень: прод ставит
  `X-Frame-Options: DENY` глобально; главная декорирована `@xframe_options_sameorigin` (грузится
  в iframe), а остальные storefront-страницы — нет → при клике в превью браузер их блокирует
  («refused to connect»), поэтому ошибка ТОЛЬКО в редакторе. Фикс: `StorefrontFrameOptionsMiddleware`
  выдаёт `SAMEORIGIN` для storefront (кабинет/логин — `DENY`; `xframe_options_exempt` G10-виджет не трогаем).
  Проверено в браузере (Pranasy): клики по ссылкам открывают страницы. (Доп. «редактирование самих
  под-страниц» — отдельно в H1/H1.2.)
- **H0 — Каркас + матрица.** 🚧 идёт. Реестр «архетип → 3 сущности + применимые секции/настройки/поля»
  + движок гейтинга видимости. **Сделано (срез 1): секция → архетип-модуль** (`archetypes.SECTION_ARCHETYPE_MODULE`
  + `section_visible_for(tenant, key)`); **гейтинг списка секций редактора** (пекарня не видит
  Stay/Events/Services/Handwerker; carry-forward конфига скрытых секций при сохранении). Проверено в браузере
  + тесты `test_archetype_gating.py`. **Дальше H0:** гейтинг storefront-меню по архетипу (пекарня не показывает
  Book/Stay/Events в шапке), реестр 3 сущностей на архетип, матрица применимых НАСТРОЕК/ПОЛЕЙ (не только секций).

### Матрица: секция главной → архетип-модуль (источник правды гейтинга)
`archetypes.SECTION_ARCHETYPE_MODULE`. Секция БЕЗ записи = **generic** (любой архетип).

| Секция | Архетип-модуль | Видна, если |
|---|---|---|
| hero, usp_bar, promotions, about, process, team, cta, testimonials, trust, reviews, faq, gallery, contact, archetypes | — (generic) | всегда |
| products, categories | catalog | catalog активен (⚠️ catalog = **core**, активен у всех → видны везде) |
| stay_search, stay_rooms | stays | stays активен |
| services | booking | booking активен |
| events | events | events активен |
| before_after | jobs | jobs активен |

⚠️ **Открытое решение (catalog core):** products/categories видны у ВСЕХ архетипов (catalog нельзя выключить).
Для «отель не показывает Товары» нужно либо сделать catalog опциональным, либо гейтить по *primary*-архетипу,
а не только `is_module_active`. Отложено до согласования — пока over-inclusion безопасен (магазин-секции пусты → не рендерятся).
