# Концепт «Живая продажа + Finder»: покупка нового поколения (эра ИИ, без ИИ в v1)

> Создан 2026-07-18 по запросу владельца: «эра ИИ — предложить совершенно новое
> не только по созданию сайта, но и по продаже/ведению/обслуживанию клиента:
> мгновенные продажи, может стримы, видеозвонок продавцу прямо с сайта,
> прозрачность и доверие; для покупателя — не часами выбирать, а как с ИИ:
> сказал, что нужно → получил рекомендации, но без ИИ, на заданных параметрах;
> может, уйти от категорий; новая форма получения результата и заказа».
> Сестринский док: `studio-concept-2026-07-18.md` (как СТРОИТСЯ сайт).
> Этот — как ПОКУПАЮТ и как продавец ОБСЛУЖИВАЕТ. Статус: КОНЦЕПТ НА ОБСУЖДЕНИЕ.
> Семейства ID: **LS** (Live-Selling), **FD** (Finder).

## 1. Тезис

Наше нечестное преимущество перед Wix/Jimdo и перед ИИ-стартапами: у нас уже
есть ОПЕРАЦИОНКА бизнеса (запись, заказы, сметы, чат, оплаты, клиенты). Значит,
«совершенно новое» мы собираем КОМПОЗИЦИЕЙ готовых движков, а не стройкой с нуля:

1. **Finder** — покупатель говорит, что ему нужно (2–4 вопроса-чипа), и получает
   3 готовых предложения с кнопкой действия — вместо блуждания по категориям.
   «Как ИИ, но без ИИ»: правила поверх наших же структурированных данных
   (атрибуты, коллекции, фасеты, цены, наличие). ИИ-ready: позже LLM садится на
   тот же контракт «интент → фильтры → предложения» без переделки.
2. **Live-Beratung** — видеоконсультация с продавцом прямо с сайта: по записи
   (реюз booking) или «сейчас на месте — позвонить». Показать торт/ткань/машину
   по видео. Персонально (1:1), не «стрим на толпу» — см. рынок §2.
3. **Sofort-Angebot** — мгновенная продажа из разговора: продавец в чате
   собирает персональное предложение (позиции+цена) → покупателю падает
   ссылка «оплатить в 1 клик». Реюз jobs.Angebot (принять/отклонить смету уже
   работает) + Stripe.
4. **Слой доверия** — «Jetzt im Laden» (живое присутствие), время ответа,
   команда с фото, верифицированные отзывы (есть), верификация в агрегаторе.

Формула покупателя: **Сказал, что нужно → увидел 3 варианта → спросил живого
человека (если хочет) → оплатил в 1 клик.** Ни одна платформа сегмента не
закрывает эту цепочку целиком.

## 2. Рынок (веб-рисёрч 07.2026 — заполняется из отчётов сессии)

- **Live-стримы на толпу — НЕ наш v1 (подтверждено)**: Instagram Live Shopping
  закрыт 03.2023, Facebook Live Shopping — 10.2022; TikTok Shop в Германии
  (старт 03.2025) к маю купили лишь ~2,5 % немцев; даже в TikTok Shop US Live
  даёт только 14 % GMV; Whatnot ($8 млрд GMV 2025) — фанатские ниши с
  аукционным азартом. Лайв-конверсии 9–30 % — но по САМООТОБРАННОЙ аудитории
  стрима, которой у пекарни нет. Стримы-события — v2 по спросу.
- **1:1 видео-шопинг — работает, но как ОПЦИЯ по записи**: Bambuser one-to-one
  (Elkjøp 30 % CR; средние 12,4 % CR, 3× против статичного сайта, +225 %
  add-to-cart); клиентелинг-заказы AOV +79 %. Красный флаг тяжёлых решений:
  Klarna купила Hero за ~$160 млн (2021) и продала Bambuser за ~€1,3 млн
  (2024) — в ~100× дешевле. Вывод: НЕ строить платформу видеошопинга — дать
  лёгкую опцию «видео-показ по записи» на нашем booking для дорогих/
  индивидуальных позиций (торт на свадьбу, ателье, мебель, номер отеля).
- **Продажа в переписке — реальный паттерн DACH**: WhatsApp у 94 % населения
  DACH; нативного WhatsApp-чекаута в ЕС нет и не ждут до ~2027 → микро-бизнес
  реально «согласовал в чате → payment-link (Stripe/SumUp) → оплата». Наш
  Sofort-Angebot — тот же паттерн, но в НАШЕМ чате: история клиента (CRM),
  наш чекаут, заказ в канбане, право DACH.
- **Право DE на видео**: запись разговора без согласия — §201 StGB (уголовно).
  v1 — БЕЗ записи звонков вообще; провайдер с EU-хостингом (Whereby Embedded
  не хранит записи; Jitsi self-host; Daily Prebuilt).
- **Доверие (BrightLocal 2024)**: 50 % доверяют отзывам как личным
  рекомендациям; 55 % лучше относятся к бизнесу, когда владелец ОТВЕЧАЕТ на
  отзывы; отзывы на странице дорогого товара — до +380 % CR. Наш слой доверия
  (LS-4) опирается на уже построенные отзывы+ответы — добавить видимость.
- **Guided selling / finders — цифры (рисёрч 07.2026, источники в отчёте
  сессии)**: RevenueHunt-бенчмарк (45M ответов): **5,5 % прошедших квиз
  оформляют заказ** (~2,75× типичного магазина с 2 % CR), **AOV +11–15 %**;
  кейсы: Crobox/Love Stories **CR +604 %**, ASICS ROI 16×, **Einhell (DACH,
  инструменты) +60 % продаж за 3 мес**, Polysleep 6× CR. Оптимум — 5–8
  вопросов у больших, для микро-каталога достаточно 3–4 чипов; **выдача — 3
  варианта, средний помечен «Unser Vorschlag»** (правило трёх + дефолт-эффект:
  бейдж «Recommended» +26 %).
- **Психология**: jam study (24 сорта → 3 % покупок, 6 сортов → 30 %);
  мета-анализ 2010 уточняет — overload бьёт именно НЕэксперта в сложной
  категории, т.е. ровно нашего покупателя. Пользователи поиска по сайту ≈3×
  чаще покупают, чем «ходоки по категориям»; intent-навигация: кейс +16 % CR.
  Прямых данных за ПОЛНЫЙ отказ от категорий нет → гибрид: intent-first вход,
  категории — fallback «Alle ansehen».
- **Формат**: диалоговые формы завершают 47,3 % против 21,5 % у классических —
  но ЧЕКАУТ должен оставаться коротким одностраничным (подбор — диалогом,
  оплата — без визарда). Ровно так и проектируем: Finder-диалог → наш обычный
  быстрый чекаут.

## 3. Finder (FD) — «результат вместо каталога»

### Механика
- **Дерево вопросов per-архетип** (из коробки, правится в кабинете):
  - Bäckerei: «Anlass?» (Geburtstag/Feier/Alltag) → «Für wie viele?» → «Wann
    abholen?» → 3 предложения (торт-вариант, набор, комбо) + «Vorbestellen».
  - Metzgerei: «Was ist geplant?» (Grillen/Party/Alltag) → «Personen?» →
    Partyservice-пакеты + заявка.
  - Friseur: «Was möchtest du?» (Schnitt/Farbe/Styling) → «Wann?» → 3 услуги
    со свободными слотами + «Buchen».
  - Hotel: даты → «Wer reist?» → 3 номера (наш поиск уже так работает — H2!).
  - Werkstatt/Handwerker: «Was ist passiert?» → срочность → запись/заявка.
- **Ответ = фильтр**: каждый вариант ответа мапится на существующие механизмы —
  коллекцию (M2M `collections`), фасет (FacetProvider), атрибут, ценовой
  диапазон, наличие слотов. НОВОГО хранилища данных не нужно; конфиг finder'а —
  ключ `site_config["finder"]` (вопросы→маппинги), normalize-ключ добавить.
- **Выдача = 3 карточки** (контракт SellableEntity уже единый): средняя —
  «Unser Vorschlag» (дефолт-эффект), у каждой — прямой CTA по purchase_mode
  (в корзину / записаться / забронировать / заявка). Кнопка «Alle ansehen →»
  ведёт в классический листинг с уже применёнными фильтрами (фасеты живы).
- **Где живёт**: (а) hero-виджет главной «Was suchst du?» — чипы первого
  вопроса прямо в баннере (новая секция `finder` в реестре SECTIONS);
  (б) страница `/finder/`; (в) в агрегаторе на городских страницах (v2) —
  «найди пекаря под задачу» — усиливает нашу платформенную петлю.
- **Категории не умирают** — они уходят на второй план (навигация для тех, кто
  хочет листать). Дефолтная главная архетипа начинается с интента, не с сетки.

### Волны FD
| ID | Волна | Объём |

| FD-2 | Пресеты деревьев на 14 архетипов + демо-киты + секция «finder» на главной (hero-чипы) | M |

## 4. Live-Beratung + Sofort-Angebot (LS) — живая продажа

### Механика (всё — композиция готовых движков)
1. **LS-1 Video-Beratung по записи**: `booking.Service` получает флаг
   `is_video` → бронь как обычно (слоты/подтверждение/напоминания уже есть),
   в письме/ЛК — ссылка на видеокомнату. Провайдер v1 — **self-hosted Jitsi**
   на нашем Hetzner (GDPR: свой сервер в DE, без аккаунтов, комната на бронь)
   ИЛИ простой фолбэк «WhatsApp-Video» (DACH-любимец) — решение §6.2. Кнопки:
   на детали товара «📹 Per Video zeigen lassen», на сайте «Live-Beratung».
2. **LS-2 «Jetzt erreichbar»**: тумблер присутствия у владельца (или авто по
   часам работы + ручной override) → на витрине бейдж «Jetzt im Laden —
   Video-Anruf möglich» и кнопка мгновенного звонка (та же комната). Не отвечает
   30 сек → фолбэк «оставить сообщение» (inbox) или «записаться» (booking).
3. **LS-3 Sofort-Angebot**: в чате (inbox) продавец жмёт «Angebot senden» →
   собирает позиции (пикер по Angebot-гриду FB-8) + цена/срок → покупателю
   приходит ссылка-предложение (реюз публичной сметы jobs `/angebot/<token>/`
   «принять/отклонить», обобщить с jobs на розницу/услуги) → оплата Stripe в
   1 клик, заказ падает в канбан. «Мгновенная продажа» из разговора.
4. **LS-4 Слой доверия**: «время ответа ~15 мин» (считаем по inbox), фото
   команды на карточках (team-секция есть), «Jetzt geöffnet» (часы есть),
   бейдж верификации в агрегаторе; счётчики «продано на этой неделе» (по
   заказам) — опционально, решение владельца.

### Почему НЕ стримы на толпу (v1)
Один пекарь не может вести регулярные эфиры; западные платформы массовых
live-shopping закрылись/нишевые. Наша форма — «живой продавец в один клик,
когда покупателю нужно» — операционно посильна и отличает нас сильнее.
Стримы (анонс эфира → страница с товарами эфира) — v2 по спросу, тем же
Jitsi/страницей события (events-движок есть).

### LS-5 Care-цикл: забота КАК ЧАСТЬ продажи (запрос владельца 2026-07-18)
Каждая сделка окружена автоматическими «человеческими» касаниями; большинство
кирпичей построено — свести в единый цикл с лицом живого человека:
- **До**: напоминания о записи/брони/оплате (есть, B2/beat) + «мы готовим ваш
  заказ» (статусы канбана видны клиенту в ЛК — есть).
- **Момент**: подтверждение с подписью живого человека — «Maria hat deine
  Bestellung angenommen» (имя+фото из team-секции), а не безликое «no-reply».
- **После**: post-purchase/post-visit «Wie war's?» (письма уже есть) →
  **развилка настроения**: 👍 → просьба публичного отзыва (есть);
  👎 → НЕ отзыв, а прямая линия к владельцу (LS-6, service recovery).
- **Долгосрочно**: штампы/лояльность (есть), авто-win-back (есть, B4),
  день рождения/годовщина (v2).
- **Управление**: тумблеры «касаний заботы» в Marketing-хабе — расширение
  готовой матрицы уведомлений UD4-2 (`site_config["notify"]`).

### LS-6 «Прямая линия»: если что-то пошло не так — сразу к владельцу
Цель: недовольство перехватывается ЛИЧНО до публичного отзыва, клиент
чувствует, что его слышит хозяин, а не «поддержка».
- Кнопка **«Etwas stimmt nicht?»** на странице заказа/брони/подтверждения,
  в письмах и в ЛК клиента → приоритетный тред в inbox С КОНТЕКСТОМ сделки
  (ссылка на заказ/бронь), флаг «⚠️ Problem».
- **Пуш владельцу немедленно** — Telegram-канал владельца уже построен
  (UD4-2 `send_to_owner`); в кабинете «Problem»-бейдж поверх всего, на
  карточке канбана — красная полоса. Для команд — назначение ответственного
  (Membership-роли есть).
- **SLA-таймер**: время реакции на проблему считается; хорошая метрика
  становится ПУБЛИЧНЫМ бейджем доверия («antwortet in ~15 Min», LS-4).
- **Service recovery**: тред решён → мягкое «Jetzt alles gut? Магst du das
  teilen?» → просьба отзыва. (BrightLocal: 53 % ждут ответа на негатив в
  течение недели; 55 % лучше относятся к бизнесу, отвечающему на отзывы.)
- «Ближе»: в ЛК клиента — «Dein Ansprechpartner: Maria» + прямой чат +
  живые статусы; у продавца — карточка клиента 360° (CM-8, есть).

## 5. Как это сшивается со Studio-концептом

- Finder-виджет и Live-кнопки — просто новые СЕКЦИИ/БЛОКИ в реестрах → сразу
  получают варианты отображения (L3) и входят в Look'и архетипов (L1).
- Marketing-хаб получает «Live»-раздел: расписание консультаций, конверсия
  Angebot-ссылок, ответ-время.
- Канбан: видеоконсультация — обычная бронь; Sofort-Angebot — карточка сделки.
- Мастер онбординга: слайд «Wie verkaufst du?» (вкл. Finder/Live) — v2.

## 6. РЕШЕНИЯ ВЛАДЕЛЬЦА (2026-07-18, «действуй автономно»)

1. **Порядок треков: FD первым** → LS-3 (Sofort-Angebot) → LS-1/2 (видео).
2. **Видео-провайдер v1: WhatsApp** («в» — WhatsApp-Video кнопка, ноль ops,
   94 % DACH; свой Jitsi — вторым шагом; БЕЗ записи §201 StGB).
3. **Finder на главной: сначала ОПЦИЯ** (в Look'ах/секции по выбору владельца
   сайта), дефолт — после обкатки.
4. Счётчики доверия («продано N…») — решение отложено (спросить при LS-4).

## 7. Риски

- Видео: право DE (согласие, никакой записи в v1; провайдер в DE), нагрузка
  на владельца (присутствие — только явный тумблер, никаких ожиданий 24/7).
- Finder: качество маппингов = качество выдачи → пресеты архетипов обязаны
  быть осмысленными из коробки; пустая выдача → фолбэк «3 популярных + чат».
- Sofort-Angebot: не плодить второй чекаут — оплата строго через существующие
  order/booking-пути (единый учёт, канбан, склад).
- normalize: новый ключ `finder` добавить в схему осознанно (golden-замки).
