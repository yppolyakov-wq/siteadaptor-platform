# Roadmap — следующие спринты (после «Витрина 2.0»)

Рабочий документ-чеклист. Идём сверху вниз, один PR на задачу, CI зелёный → мерж →
деплой через `./scripts/deploy.sh single`. Миграции — строго последовательные.

Ссылки на паттерны: `docs/references/patterns/` (state-machine, anti-oversell,
notification-dedupe, audit-log, soft-delete, cursor-pagination, webhook-hmac).

## Track A — Завершить инструменты акций

### A2. Карта лояльности (штампы)
**Цель:** удержание оффлайн-клиентов («10-й кофе бесплатно»).

- [ ] A2.1 Решения (опрос перед стартом)
  - [ ] награда: авто-генерация `Voucher` по `reward_label` **или** просто отметка «reward готов»
  - [ ] идентификация клиента: email/телефон **или** обязательный персональный loyalty-QR
- [ ] A2.2 Модели (миграция `promotions/0007`)
  - [ ] `LoyaltyProgram`: label, `stamps_required`, `reward_label`, `is_active`, опц. scope (товар/категория/общая)
  - [ ] `LoyaltyCard`: FK Customer + FK Program, `stamps`, `rewards_earned`, unique(customer, program)
  - [ ] (опц.) `StampEvent` — лог начислений (аудит + анти-дабл)
- [ ] A2.3 Сервисы
  - [ ] `add_stamp(program, customer)` — атомарный `+1`; при достижении порога → награда + сброс остатка
  - [ ] анти-дабл (кулдаун/идемпотентный ключ на визит)
- [ ] A2.4 Кабинет: CRUD программ; экран «начислить штамп» (поиск/скан карты → прогресс → +1); список карт
- [ ] A2.5 (опц.) Клиентская страница «моя карта» по токену + QR
- [ ] A2.6 DSGVO: карты на Customer → попадают под авто-очистку PII
- [ ] A2.7 Тесты: начисление, порог→награда+сброс, анти-дабл, безлимит/повтор
- **DoD:** программа создаётся; штамп начисляется идемпотентно; при пороге выдаётся награда; покрыто тестами.

### A3. Аналитика акций
**Цель:** владелец видит, что работает.

- [ ] A3.1 Решение: MVP-счётчики **или** дневные агрегаты для графиков
- [ ] A3.2 Сбор: счётчик `views` на `promotion_detail` (атомарный `F()+1`); брони из `Reservation`; каналы из `source_channel`; waitlist/ваучеры — count
- [ ] A3.3 Метрики на акцию: просмотры, брони по статусам, **конверсия** = брони/просмотры, **выдачи** (fulfilled) и % выкупа, топ-каналы
- [ ] A3.4 UI: блок «Статистика» на форме акции (рядом с таблицей каналов) + сводная страница **Analytics** в сайдбаре
- [ ] A3.5 (опц.) `PromotionDailyStat` — дневные агрегаты + графики (период 7/30 дн.)
- [ ] A3.6 Тесты: инкремент просмотров, конверсия, разбивки
- **DoD:** на акции видно просмотры/брони/конверсию; сводная страница со всеми акциями.

## Sprint 4 — Авто-публикация и локальный агрегатор
**Цель:** активная акция автоматически попадает в каналы и в локальный агрегатор (то,
ради чего заложен seam `AggregatorPortal`). Мульти-домен агрегатора — Phase 2.

- [ ] S4.1 Приложение `apps.publishing` (TENANT)
  - [ ] `Channel`: тип (aggregator/website/…), включён, конфиг
  - [ ] `Publication`: promotion FK, channel, status (FSM: queued→published→failed/removed), external_ref, idempotent `dedupe_key`
- [ ] S4.2 Хуки FSM акции (есть в state-machine.md): `active` → `publish_to_channels_task.delay(...)`; `ended` → снять листинг
- [ ] S4.3 Celery-задачи (idempotent_task + dedupe): публикация/снятие; ретраи; запись `Publication.status`
- [ ] S4.4 Локальный агрегатор (`apps.aggregator`, SHARED-чтение по индексам city/business_type)
  - [ ] материализация активных акций в листинг; страница агрегатора (город/категория)
  - [ ] `sync_aggregator_listing` по переходам акции
- [ ] S4.5 Кабинет: тумблеры каналов на акции; статус публикаций
- [ ] S4.6 Тесты: публикация при активации, снятие при ended, идемпотентность, агрегатная выборка
- **DoD:** активация акции → видна в агрегаторе; завершение → исчезает; повтор задачи не дублирует.

- [ ] S5.1 dj-stripe: продукты/цены (39 €/мес), Customer/Subscription синхронизация

## Рабочее соглашение
- Один PR на задачу; ветка `claude/<кратко>`; CI (`lint-and-test`) зелёный → squash-merge.
- Миграции последовательные; стек-ветки ребейзить на свежий main перед PR.
- После пачки мержей с миграциями — деплой: `git pull origin main && ./scripts/deploy.sh single`.
- Тесты на django-tenants: вьюхи — через RequestFactory; новые TENANT-приложения видны в `config/settings/test.py` автоматически.
