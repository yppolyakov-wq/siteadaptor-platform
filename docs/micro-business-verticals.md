# Микробизнесы DACH — карта вертикалей, потребностей и «логической полноты»

Назначение: единый справочник, для каких микробизнесов Германии платформа полезна,
что им нужно в дневном цикле, что у нас уже есть и чего не хватает до «логического
завершения». Source of truth для планирования вертикалей (дополняет
`full-platform-vision.md` §2/§5 и `roadmap-next-sprints.md`).

Составлено: 2026-06-12 (сессия анализа экосистемы). Держим в синхроне с реестром
модулей `apps/core/modules.py` и фактом в `main`.

## 0. Принцип: вертикали = 3 «движка», а не N продуктов

Внутри одного движка потребности почти совпадают, поэтому «полнота» достигается
не по одному бизнесу, а одним пакетом общих фич сразу для группы.

| Движок | Кто сюда попадает | Статус ядра |
|---|---|---|
| **Retail / каталог** | магазины (еда, непрод, handmade) — масса микробизнеса | ✅ есть, не хватает 4 деталей |
| **Booking по времени** (time-slot) | салон, столы, слот-услуги/курсы | ✅ почти готов |
| **Booking по датам** (ночи) | гостиница, ретрит, Ferienwohnung | ✅ есть (Track E / `apps.stays`) |

Спектр, который платформа должна покрыть: **локальный магазин → интернет-магазин
(Versand) → услуги по записи → гастро → размещение → события → выездной сервис →
агрегатор/маркетплейс.**

## 1. Архетипы и их «логическая полнота»

Покрытие (%) — грубая оценка готовности дневного цикла бизнеса на платформе.

### A1. Laden / розничный магазин (каталог + самовывоз + локально) — ~80%
**Кто:** Bäckerei, Konditorei, Metzgerei, türkischer/orientalischer Gemüse-Obst-Basar,
Lebensmittel, Asia-Markt, Afro-/Ost-Shop, Teeladen, Kaffeerösterei, Käse-/Weinhandlung,
Hofladen, Feinkost, Blumenladen, Buch-/Schreibwaren, Boutique/Schuhe, Spielwaren,
Unverpackt-Laden, Kiosk/Späti.
**Цикл:** ассортимент → дневные акции/Überraschungstüte → бронь/предзаказ на
самовывоз → лояльность → найти в городе.
**У нас:** `catalog` ✅, `promotions`(+surprise) ✅, `orders`(Click&Collect) ✅,
лояльность ✅, витрина (`tenants.siteconfig`) ✅, `aggregator` ✅, `finance`(Rechnung,
НДС 7/19%) ✅.
**До полноты:** варианты товара (G2), весовая/единичная цена + Grundpreis €/кг (G2),
реальный остаток на товаре (G3), онлайн-оплата (G1), аллергены/LMIV (G7).

### A2. Интернет-магазин / Versand (чистый онлайн) — ~75% (доставка ✅ G4)
**Кто:** Direktvermarkter (vegane Wurst, Honig/Imkerei, Hofprodukte),
Kaffeerösterei-Versand, Handmade/Etsy-тип (Schmuck, Keramik, Kerzen, Seife,
Stricksachen), Nahrungsergänzung, kleine Manufaktur.
**Цикл:** каталог → корзина → **доставка по адресу** → онлайн-оплата → счёт/трек.
**У нас:** каталог + корзина (заточена под самовывоз) ✅ частично.
**У нас:** ✅ доставка/Versand (G4): адрес, плоский тариф + бесплатно-от +
Mindestbestellwert + зона-текст, статус «versandt» + трек-номер, оплата (P2.5c).
**До полноты:** зоны по PLZ-таблице, лейбл через API перевозчика + Lieferschein,
возвраты/Widerruf — отложено (см. roadmap).

### A3. Termin / услуга по записи (time-slot) — ~85%
**Кто:** Friseur, Barber, Kosmetik/Nagel/Wimpern/Waxing, Massage, Fußpflege/Podologie,
Hundesalon, Tattoo/Piercing, Fotograf, Fahrschule, Nachhilfe/Musikschule, Yoga/Pilates,
Werkstatt-Termine (KFZ/Reifen/Handy/Änderungsschneiderei), Coach/Berater.
**У нас:** `booking` (ресурс/услуга/слот) ✅, календарь, напоминания ✅.
**До полноты:** депозит/предоплата (G1), абонемент/Mehrfachkarte + групповой курс с
вместимостью (G9; capacity уже есть), no-show-защита.
**⚠ Ограничение:** Massage/Heilpraktiker/Physio — Heilberuf: DSGVO-чувствительные
данные + Heilmittelwerbegesetz. Берём только «запись», без медкарт.

### A4. Gastro (меню + столы + доставка) — ~65%
**Кто:** Restaurant, Café, Bistro, Imbiss, Eisdiele, Bar, Shisha-Bar, Foodtruck,
Bäckerei-Café.
**У нас:** меню=`catalog` ✅, бронь столов=`booking`(time-slot) ✅, самовывоз ✅.
**До полноты:** доставка (зоны/сбор/мин-сумма) (G4-вариант), модификаторы блюда
(extras), кухонная очередь/KDS (опц.), депозит за стол (G1), аллергены (G7).

### A5. Übernachtung / размещение (date-range) — ~80% (движок ✅ Track E)
**Кто:** Pension, kleines Hotel, Ferienwohnung/-haus, B&B, Gästezimmer, Hostel,
Campingplatz/Stellplatz.
**У нас:** ✅ `apps.stays` (Track E / G5): инвентарь по-ночам (`StayUnit.quantity`),
календарь загрузки, тариф/ночь, поиск по диапазону, anti-double-book на интервал
(`book_stay`), витрина `/unterkunft/`, депозит онлайн (Stripe Connect), письма +
напоминание, выручка в `finance` (НДС 7 %). Гость=Customer переиспользован.
**До полноты:** сезонные/выходные тарифы, листинг размещений в агрегаторе,
авто-Rechnung на бронь, iCal/каналы (Booking.com) — отложено (см. roadmap).

### A6. Событие / ретрит (дата + вместимость + билет/программа) — ~45–55%
**Кто:** Seminarhaus/Retreat, Yoga-Retreat, Kochkurs/Töpferkurs/Malkurs,
Stadtführung/Exkursion, Tour-Operator, kleine Konzerte/Events.
**У нас:** слот с датой+вместимостью ✅ (`Resource.capacity` + `Booking.party_size`),
билет≈бронь/ваучер+QR ✅, waitlist ✅.
**До полноты:** платный билет (G1), список участников/анкеты, ретрит — date-range
(G5) + микромодуль «программа», тур — гид/транспорт.

### A7. Выездной Handwerker (заявка → смета → работа → счёт) — ~75% (движок ✅ G6)
**Кто:** Maler, Elektriker, SHK/Klempner, Fliesenleger, Schreiner,
Garten-/Landschaftsbau, Gebäudereinigung, Hausmeisterservice, Umzug, Schlüsseldienst,
Catering, mobiler Friseur, Schädlingsbekämpfung.
**Цикл (другой!):** Anfrage → **Angebot/Kostenvoranschlag** → Termin/Auftrag → Rechnung.
**У нас:** ✅ `apps.jobs` (G6): форма заявки на витрине `/anfrage/`, конструктор сметы
+ Angebot-PDF, публичное принятие сметы `/angebot/<token>/`, статусы (JobSM),
«Rechnung erstellen» из позиций (finance.Invoice), письма. Контакт=`crm` ✅.
**До полноты:** Anzahlung онлайн за смету (Handwerker платят по счёту — отложено),
дробные часы/единицы в позициях, фото к заявке, привязка Termin под работу (booking).

### A8. Агрегатор / Marktplatz (дальний конец спектра)
**Что:** городской портал, где все архетипы находятся, а потребитель сам
ищет/бронирует/заказывает/покупает по разным бизнесам.
**У нас:** `aggregator` + мультидоменные порталы + клиентские аккаунты (magic-link) +
избранное + featured ✅.
**До полноты:** оплата из портала (G1), отзывы/рейтинги (G8), гео-поиск/карта (G8),
единая корзина по бизнесам (тяжёлое, опц.).

## 2. Сквозной бэклог: что закрывает несколько архетипов

| Код | Фича | Закрывает | Леверидж |
|---|---|---|---|
| **G1** | Онлайн-оплата (P2.5) | A2,A4,A5,A6,A8 + депозиты A3 | 🔥 максимум |
| **G2** | Варианты + весовая цена + Grundpreis (PAngV) | A1,A2 | масса розницы |
| **G3** | Реальный остаток на товаре/заказе | A1,A2,A4 | розница+гастро |
| **G4** | Доставка/Versand (адрес, тарифы, лейбл) ✅ (`apps.orders`) | A2 (+доставка A4) | «настоящий магазин» |
| **G5** | Date-range booking ✅ (Track E, `apps.stays`) | A5,A6 | семья «размещение» |
| **G6** | Anfrage→Angebot→Auftrag (смета) ✅ (`apps.jobs`) | A7 | все Handwerker |
| **G7** | Аллергены/LMIV + Herkunft | A1,A4 (еда) | правовое для еды |
| **G8** | Отзывы/рейтинги + гео-карта | A8 (+доверие всем) | завершает агрегатор |

## 3. Порядок завершения по левериджу (предложение, не план)

1. **G1 онлайн-оплата** — раскрывает доставку, депозиты, билеты, чекаут портала.
2. **G2+G3+G7 retail-пакет** — завершает массу магазинов (чай, турецкий, пекарня,
   мясная, бутик) — большинство уже на платформе.
3. **G4 доставка** — добивает чистый интернет-магазин (A2).
4. **G5 date-range** — открывает гостиницы/ретриты (A5/A6).
5. **G6 смета** — открывает ремесленников (A7, новый сегмент).
6. **G8 отзывы+гео** — доводит агрегатор до маркетплейса (A8).

## 4. Текущее покрытие платформой (куда смотреть в коде)

- Каталог/витрина: `apps/catalog`, `apps/tenants/siteconfig.py`, `apps/imports`.
- Акции/брони/лояльность/ваучеры/QR: `apps/promotions`.
- Click&Collect: `apps/orders`.
- Запись по времени: `apps/booking`.
- Финансы/Rechnung/DATEV: `apps/finance`.
- CRM-мини: `apps/crm`.
- Агрегатор/порталы/клиентские аккаунты: `apps/aggregator`.
- Каналы (GBP): `apps/publishing`. Уведомления: `apps/notifications`.
- Подписка/featured: `apps/billing`. Реестр модулей кабинета: `apps/core/modules.py`.

## 5. Ограничения и развилки (зафиксировать перед стартом)

- **Heilberuf** (массаж/Heilpraktiker/Physio/медицина): особые DSGVO-данные +
  Heilmittelwerbegesetz — без медкарт, только запись.
- **LMIV** (аллергены) и **PAngV/Grundpreis** (€/кг) — для еды это закон, не UX:
  драйвер фич G7/G2.
- **«39 € без комиссий, без трекинг-куки, DSGVO-first»** конфликтует с
  маркетплейс-комиссиями/lead-gen/рекламой (A8 «тяжёлый»). Решение о монетизации
  маркетплейса — отдельно (см. roadmap §P2.4/Отложено).
- **Фокус vs ширина:** ТЯЖЁЛЫЙ ERP (склад/производство/закупки/логистика) и внешние
  маркетплейсы (Amazon/eBay) — НЕ из этих микробизнес-вертикалей; вне периметра
  данного документа.

## 6. План работ (принято владельцем 2026-06-12)

Очередь исполнения (каждая подзадача — отдельный инкремент, ветка → CI зелёный →
чекпоинт, как P2.4b):

1. **R4 — Allergene/LMIV + Herkunft** (G7) — *в работе.* Структурные поля товара
   (14 аллергенов EU, Herkunft, Zutaten) + вывод на витрине. Независим, без оплаты.
2. **P2.5 — Онлайн-оплата** (G1) — депозит брони/термина, оплата Click&Collect;
   разблокирует доставку, билеты, чекаут портала. Фундамент Stripe готов (подписка
   + featured P2.4b).
   - **Модель денег (решение 2026-06-13):** Stripe Connect, аккаунты **Standard**
     (онбординг через OAuth). Клиент платит бизнесу 100 % напрямую. Монетизация —
     **вариант B**: комиссию (% по типу бизнеса, `BILLING_APPLICATION_FEE_PERCENT`,
     сейчас 0) выставляем продавцу как «Nutzungsgebühr» строкой в счёте подписки,
     а не удерживаем в платеже. Вариант A (`application_fee` в Stripe) — хук в коде
     на будущее; вариант 3 (платформа собирает + payout) — резерв под маркетплейс.
   - **Подзадачи:** P2.5a онбординг (поля Tenant + OAuth-сервис + вебхук
     `account.updated` + кабинет) → P2.5b депозит брони/термина → P2.5c предоплата
     C&C → P2.5-fee метрика оборота + строка Nutzungsgebühr в инвойс (вариант B,
     при ненулевом %).
3. **R1 → R2 → R3** — остаток retail-пакета: варианты товара → Grundpreis/единичная
   цена (PAngV) → реальный остаток с atomic-списанием. Завершает архетип A1.

Разбивка retail-пакета (R1–R4) — см. чат-разбор 2026-06-12 / при старте каждой
подзадачи фиксируем детали в CLAUDE.md §3.
