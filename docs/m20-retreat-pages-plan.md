# M20 (Phase 3) — Ретрит-страницы + универсальный layout-движок · план

Статус: 🚧 план (2026-06-24). Развитие Site Builder (M20). Принцип прежний —
**без новых моделей**, всё поверх JSON `Tenant.site_config` + реестр секций
(`apps/tenants/siteconfig.py`). Анти-Битрикс: пресеты вперёд, настройки спрятаны,
детальные страницы — управляемый скелет, а не чистый холст.

## Решения владельца (2026-06-24)
1. **Флоу покупки ретрита — 2-шаговый чекаут** (без корзины): Шаг 1 — тариф/номер/
   extras + цена → «Weiter»; Шаг 2 — контакты + waiver + оплата (вкл. рассрочку R10).
   Корзина остаётся магазину мерча (`/warenkorb/`). Модель не меняется — `book_ticket`
   по-прежнему один атомарный вызов в конце.
2. **Первый срез — ретрит-страницы** (обзор + детальная + чекаут). Layout-движок
   поднимаем здесь как переиспользуемый фундамент.
3. **Глубина настроек — пресеты + «Дополнительно»**: пресет раскладки одним кликом
   (Список / 2 / 3 / 4 в ряд / Галерея), ручные `cols/mobile/gap` — в свёрнутом блоке.

## Контекст из кода (карта 2026-06-24)
- Раскладка **захардкожена** в каждом `templates/storefront/sections/_*.html`
  (`grid-cols-*`); `_promotions.html` уже имеет свой мобильный 1/2-тоггл — паттерн назрел.
- Per-page конфигурации нет (только главная); но `site_config["archetypes"][key]`
  (intro/hero/gallery) уже применяется к индексам `/veranstaltung/` и `/sortiment/`
  — фундамент под per-page.
- Источник данных секций захардкожен (products `[:8]`, reviews `6`).
- Детальная ретрита (`event_detail.html`) сейчас **1 колонка** (`max-w-3xl`) — редизайн.
- Бронь уже шлёт тариф+номер+extras+ваучер+рассрочку **одним POST** → 2 шага = UI-сплит.

## Архитектура layout-движка (фундамент, переиспользуется везде)
Расширяем элемент секции `{key, enabled}` → `{key, enabled, layout, source}`:
```
layout = { preset: list|cols2|cols3|cols4|gallery,  # пресет
           width: contained|full,                   # контейнер vs во всю ширину
           cols: 1..5, mobile: 1|2, gap: sm|md|lg }  # «Дополнительно» (override пресета)
source = { type: all|featured|category|manual, limit: int, category: slug }
```
- **Tailwind purge-safe**: один тег `{% grid_classes layout %}` (templatetags) с
  **статической таблицей** классов (никаких динамических строк) → отдаёт готовый
  responsive-класс. Заменяем им захардкоженные гриды.
- `siteconfig.normalize` расширяем: до-заполнение `layout`/`source` дефолтами
  (back-compat: старый конфиг без них получает дефолт пресета секции).
- Пресет → конкретные cols/mobile (Список=1, 2/3/4 в ряд, Галерея=4+ tight). Ручные
  значения переопределяют пресет.

## Унификация архетипов (видение владельца, 2026-06-24) — расширяет M20R

Ключевая идея: **унифицировать страницы для ВСЕХ архетипов**, а не делать
ретрит-частные. Тогда M20R становится M20U («unified»):

**Модель «архетип = главный товар + категории + способ покупки».**
- Главный товар по архетипу: магазин → товар; ретрит → ретриты; магазин без
  товаров → акция; отель → номера. (Регистр `primary_item` на архетип.)
- У каждого — категории/подкатегории (+ теги для визуализации).
- Покупаемые (товар/номер/билет) → блок «корзина/оформить»; «под запрос» →
  форма быстрого запроса. (Регистр `purchase_mode`: cart | request | booking.)

**Три унифицированных шаблона (одна семья на все архетипы):**
1. **Главная** = слайдер баннеров → категории (если есть) → главные товары/услуги/
   мероприятия → полезное (отзывы, FAQ …). Для мультисервисного тенанта (pranasy)
   главная = обобщение всех доступных услуг (как сейчас archetype-тизеры).
2. **Каталог/категория** = если есть подкатегории — сначала они (+ опц. баннер),
   затем элементы; нет категорий → общая категория. Одиночный сайт — без фильтров/
   поиска; агрегатор — с фильтрами и бо́льшим объёмом на главной.
3. **Детальная** = галерея слева (одно большое фото + миниатюры; клик по миниатюре
   меняет большое на месте) + блок цены/условий справа → подробности по теме
   (условия/даты/учителя/программа/локация/отзывы — что есть). Сходятся
   product/event/stay detail в ОДИН `storefront/detail.html` + регистр секций.

**Сквозное:** текстовые страницы (о компании…), регистрация (есть customer_account),
**чат с компанией** (на агрегаторе и внутри объекта — отдельный крупный модуль,
deferred). Настройки — в кабинет-билдере (просто), не в django-admin.

**Тест-площадка:** `pranasy.siteadaptor.de` (ретрит ⊕ ресторан ⊕ под-заказ) —
привести к общей схеме.

### Исходная разбивка M20R (поглощается M20U выше)
- **M20R-2 — обзор ретритов (категория).** Красивая страница-грид ретритов компании
  (карточка ретрита), пресет «2–3 в ряд», **фильтры/поиск OFF по умолчанию** на
  одиночном сайте (агрегатор с фильтрами не трогаем). Retreat-first: `storefront_root`
  по умолчанию → обзор ретритов для tenant-типа «ретрит».
- **M20R-3 — детальная ретрита (скелет).** Редизайн `event_detail.html`: баннер →
  **галерея слева + sticky ценовой блок справа** (2 колонки) → описание / программа /
  преподаватели / отзывы как тоггл-секции скелета (порядок/видимость из site_config).
- **M20R-4 — 2-шаговый чекаут.** Шаг 1 (тариф/номер/extras + цена, «Weiter») → Шаг 2
  (контакты + waiver + оплата/рассрочка). Прогресс между шагами — сессия/POST;
  финальный `book_ticket` без изменений. Honeypot/rate-limit сохранить.
- **M20R-5 — интеграция в билдер.** Пер-страничные настройки в кабинет-билдере:
  клик по блоку → контекстная панель (пресет/источник/контент), переключатель
  страниц (Главная/Ретриты/Шаблон ретрита), live-preview. Убрать визнастройку из admin.

## Не делаем (анти-Битрикс дисциплина)
- Не плодим модели/таблицы — только JSON.
- Детальные страницы — скелет с тогглами, не свободный холст.
- Не тащим фильтры/поиск на одиночный сайт ретрита (это работа агрегатора).
- Не даём десятки контролов на блок — пресет + свёрнутое «Дополнительно».
