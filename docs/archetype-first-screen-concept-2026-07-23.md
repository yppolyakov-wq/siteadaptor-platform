# Концепция «Задача-первым»: что стоит во главе страницы каждого архетипа

Дата: 2026-07-23. Статус: **НА СОГЛАСОВАНИЕ с владельцем** (код не пишем до
утверждения принципов и волн). Запрос владельца: «по каждому архетипу — главные
элементы, которые он позиционирует/продаёт, они должны стоять во главе страниц;
пошаговое согласование каждого процесса отображения и покупки».

## 0. Ответ на вопрос «это ли интересует пользователей?»

**Да — инстинкт верный, и он совпадает с конверсионной практикой.** Посетитель
сайта малого бизнеса приходит с ОДНОЙ доминирующей задачей (записаться /
заказать / проверить даты / узнать цену), и каждый экран между ним и этой
задачей теряет часть посетителей. Два уточнения:

1. **«Гигиенические» задачи стоят ПЕРЕД покупкой.** Для локального бизнеса
   это: «открыто ли сейчас», «сколько стоит», «где вы». Если их не видно на
   первом экране — посетитель уходит в Google Maps. Поэтому open-status,
   цены на карточках и город — часть первого экрана, а не подвала.
2. **Новый ≠ постоянный.** Новому нужно доверие рядом с кнопкой (рейтинг,
   «Verkauft N», время ответа — всё уже есть); постоянному — кратчайший
   повтор («записаться снова») — кандидат на v2 через клиентский аккаунт.

Текущий разрыв в коде: во всех 8 шаблонах сайта секции начинаются с
`["hero", …]` — hero ДЕКОРАТИВНЫЙ (текст + кнопка), а главный интерактив
(services / stay_search / events) всегда ВТОРОЙ секцией. Реестры
`PRIMARY_SECTION`/`PURCHASE_MODE`/`_PRIORITY` (apps/core/archetypes.py) уже
знают «главный товар» каждого архетипа — концепция ложится на них без ломки.

## 1. Пять принципов (утвердить один раз — действуют для всех архетипов)

- **П1. Первый экран = главное действие, интерактивное.** Hero сливается с
  primary-виджетом архетипа: не «Willkommen + кнопка», а сразу выбор
  (услуга / даты / ближайшее событие / акция с countdown). Механика: новый
  вариант секции hero «hero+primary» per-архетип; классический hero остаётся
  опцией (classic_ui и выбор владельца сайта уважаем — Р7).
- **П2. Доверие рядом с действием, а не отдельной секцией.** Рейтинг ⌀·N,
  «Verkauft N diese Woche», «Antwortet in ~N Min», open-status/«Jetzt
  erreichbar» — в пределах первого экрана, около CTA.
- **П3. ≤3 шага до денег.** Каждый шаг funnel'а задаёт ОДИН вопрос; без
  регистрации; прогресс виден; mobile — липкий buybar (есть).
- **П4. Маркетинг после действия.** Акции/лояльность/Finder/рассылки — ниже
  главного действия. Исключение — промо-архетип, где акции и ЕСТЬ действие.
- **П5. Finder — вторая кнопка, не замена.** «Не знаю, что выбрать» — опция
  рядом с главным действием, никогда вместо него.

## 2. Формат разбора

Для каждого архетипа: **Продаёт** (позиционирование) → **Задача посетителя №1**
→ **Первый экран** (что вместо/внутри hero) → **Шаги покупки** → **Вторично**
(что ниже) → **Статус в коде** (есть/гэп).

## 3. Разбор архетипов

### 3.1 Bäckerei (bakery)
- Продаёт: свежесть КАЖДЫЙ ДЕНЬ + предзаказ к времени.
- Задача №1: «что свежее сегодня, успею ли забрать к N часов».
- Первый экран: «Heute frisch» — сегодняшние хиты (Collections уже умеют
  подборки) + open-status + CTA «Vorbestellen». Цены на карточках.
- Шаги: товар → корзина → время самовывоза → оплата (on-site/online). 3 шага.
- Вторично: Wochenangebote, штампы лояльности, о нас/команда.
- Код: C&C-funnel есть; гэп — «подборка дня» первым экраном (G3) + hero-слияние (G1).

### 3.2 Metzgerei (butcher)
- Продаёт: качество мяса + **Wochenangebote** (рыночная традиция DACH!) +
  Partyservice.
- Задача №1: «что по акции на этой неделе»; №2: «заказать к выходным /
  кейтеринг на праздник».
- Первый экран: Wochenangebote (акции с ценами) + CTA предзаказ; второй
  блок — Partyservice (jobs-заявка).
- Шаги: как bakery; Partyservice: заявка → смета → принятие.
- Код: всё есть (promotions + C&C + jobs в демо metzgerei); гэп — порядок
  секций (акции первыми) + G1.

### 3.3 Lebensmittel (grocery)
- Продаёт: ассортимент рядом с домом + акции.
- Первый экран: категории-плитки (навигация = главное действие) + акции недели
  рядом + open-status.
- Шаги: категория → товар → корзина → время → оплата.
- Код: есть; вопрос только порядка секций.

### 3.4 Café (cafe)
- Продаёт: атмосферу + Karte + постоянных гостей (лояльность).
- Задача №1: «что в меню и почём, открыто ли»; №2: «забронировать столик».
- Первый экран: меню-тизер компакт-прайсом (ST-7c compact уже в демо cafe) +
  «Tisch reservieren» + open-status.
- Шаги брони: дата → время/персоны → контакт → подтверждение.
- Вторично: штампы, win-back, инста-галерея.
- Код: есть всё; гэп — G1 (бронь-CTA в первый экран).

### 3.5 Restaurant
- Как cafe, но два РАВНЫХ действия: «Speisekarte» и «Tisch reservieren»
  (вечером бронь важнее). Если включена доставка/C&C — третий вход «Bestellen».
- Первый экран: hero-фото блюда + две крупные кнопки + open-status; ниже меню.
- Код: есть; вопрос компоновки двух CTA в hero (G1).

### 3.6 Mode (clothing)
- Продаёт: стиль и новинки.
- Задача №1: «что нового, есть ли мой размер».
- Первый экран: полноширинная витрина новинок (overlay-карточки — уже в демо
  mode) + фильтр размера в один тап. Sale — отдельный вход, не первый.
- Шаги: карточка (размер обязателен) → корзина → доставка → оплата;
  per-size Warteliste (есть).
- Код: есть; гэп — размер-фильтр на первом экране (сейчас фасеты на листинге).

### 3.7 Einzelhandel (retail)
- Продаёт: «у нас это есть рядом».
- Первый экран: категории первыми (навигация), затем хиты; open-status.
- Шаги: cart-funnel. Код: есть.

### 3.8 Online-Shop
- Как retail, но БЕЗ «рядом»: доверие к доставке решает.
- Первый экран: хиты/категории + строка «Versand ab X € / Lieferzeit N Tage»
  прямо под hero + способы оплаты бейджами.
- Код: есть E-7 микс оплаты; гэп — доставка-строка в первый экран.

### 3.9 Touren (tour_operator)
- Продаёт: впечатления с датами и ограниченными местами.
- Задача №1: «куда и когда можно поехать, есть ли места».
- Первый экран: ближайшие туры с датами и остатком мест (+date-поиск);
  «N Plätze frei» = честный триггер срочности.
- Шаги: тур → дата/тир → кол-во → контакт → оплата → QR.
- Код: RV2/RV3 + тиры/депозит есть; гэп — G1 (date-поиск в hero).

### 3.10 Hotel
- Продаёт: ночи в конкретные даты.
- Задача №1: «свободно ли на МОИ даты и почём».
- Первый экран: **date-search (Anreise/Abreise/гости) поверх фото отеля** —
  слить hero и stay_search (сейчас это две отдельные секции); под ним карточки
  номеров «от X €/Nacht».
- Шаги: даты → номер (тариф/питание) → данные гостя → предоплата по тарифу
  (0/частично/100% — G7 есть).
- Вторично: отзывы, удобства, Kurtaxe/инфо, промокоды.
- Код: весь funnel есть (H1–H9); гэп — ТОЛЬКО слияние hero+поиск (G1).
  Это флагманский кейс концепции.

### 3.11 Friseur — пример владельца
- Продаёт: услугу мастера в конкретный слот.
- Задача №1: «какая услуга и почём → СВОБОДНОЕ ВРЕМЯ → (мой мастер)».
- Первый экран: список услуг с ценами и «Termin buchen» на каждой (чипы
  категорий сверху), presence-пилюля «Jetzt erreichbar», рейтинг у заголовка.
- Шаги: услуга → **[мастер — опционально]** → слот → контакт → подтверждение.
- Вторично: до/после, отзывы на детали услуги, видео-консультация (LS-1), акции.
- Код: services-секция + двухшаговый buybox + слот-пикер есть; staff-профили
  (A3) есть, но **выбора мастера в слот-пикере нет — это функциональный гэп
  (G2)**, ровно тот, что назвал владелец. Плюс G1.

### 3.12 Werkstatt
- Продаёт: решение проблемы с машиной по понятной цене.
- Задача №1: «типовая работа → цена → слот»; №2: «нетиповое → смета».
- Первый экран: плитки типовых работ с Festpreis (Inspektion / Reifenwechsel /
  TÜV) → сразу к слоту; рядом вторая кнопка «Kostenvoranschlag anfragen».
- Шаги booking: работа → слот → авто-данные (Kennzeichen/HSN/TSN — есть) →
  подтверждение. Шаги request: заявка → смета → принятие онлайн.
- Код: A9 Festpreis + anfrage есть; гэп — G1 (плитки работ первым экраном).

### 3.13 Handwerker
- Продаёт: надёжность мастера («возьмутся и не подведут»).
- Задача №1: «опишу работу → получу цену»; корзины НЕ существует.
- Первый экран: **мини-заявка из 3 полей** (что нужно + фото + контакт) прямо
  на первом экране + референсы до/после рядом (доверие = конверсия).
- Шаги: заявка → смета `/angebot/<token>/` → принятие → (оплата).
- Код: jobs-funnel целиком есть, шаблон handwerk ведёт на /anfrage/; гэп —
  G1 (форма на главной, а не переход на /anfrage/).

### 3.14 Veranstalter (events)
- Продаёт: ближайшее событие с ограниченными билетами.
- Задача №1: «что ближайшее, есть ли билеты, почём».
- Первый экран: афиша ближайшего события КРУПНО (дата + CTA Tickets +
  «N Tickets übrig») + грид следующих с countdown (RV3 есть).
- Шаги: событие → тир/кол-во → контакт → оплата → QR.
- Код: есть; гэп — G1 (афиша-режим hero).

### 3.15 Промо-архетип («у нас только акции», primary=promotions)
- Продаёт: выгоду прямо сейчас.
- Первый экран: активные акции с countdown/остатком (стили UE2 есть),
  резерв в 1 клик.
- Шаги: акция → резерв (имя+телефон, без аккаунта) → код/QR → погашение
  в магазине. Anti-oversell есть.
- Код: ядро платформы, всё есть; гэп — авто-раскладка «акции первым экраном»
  когда promotions = primary (сейчас порядок из шаблона).

## 4. Сквозные гэпы кода (реализация ПОСЛЕ согласования)

- **G1 «Интерактивный hero»** — главный гэп, общий для всех: вариант секции
  hero со встроенным primary-виджетом per-архетип (date-search / услуги /
  афиша / заявка / акции). Механика: `SECTION_STYLES["hero"]` += вариант
  «primary» + рендер по `PRIMARY_SECTION`; classic и текущие сайты не трогаем.
- **G2 Выбор мастера** в слот-пикере friseur (фильтр слотов по staff;
  staff-профили A3 уже есть). Функциональный, не только визуальный.
- **G3 «Подборка дня/недели»** для гастро (bakery «Heute frisch», butcher
  «Wochenangebote») — на базе Collections/промо, без новых моделей.
- **G4 Промо-first раскладка** — когда promotions primary, акции с countdown
  идут первым экраном автоматически.
- **G5 Гигиена первого экрана** — open-status/цены/город в hero-зоне у всех.
- **G6 Доверие у CTA** — рейтинг/sold-badge/response-time рядом с primary
  (частично есть на деталях; поднять на главную).

## 5. Процесс пошагового согласования (предложение)

Каждый шаг = «предложение → мокап на демо → чекпоинт владельца → код»:

- **Шаг 0 (сейчас):** утвердить §1 (5 принципов) и волны ниже.
- **Волна 1 — флагманы, максимально разные funnel'ы:** friseur (пример
  владельца, G1+G2) · hotel (G1: date-search в hero) · butcher/bakery
  (G1+G3 гастро). По каждому: мокап на его демо-сайте → согласование →
  реализация → следующий.
- **Волна 2:** werkstatt · handwerker · events/touren (G1 в трёх режимах:
  Festpreis-плитки / заявка / афиша).
- **Волна 3:** retail-семейство (retail/online_shop/mode/grocery) + restaurant/
  cafe + промо-first (G4).
- Замки на каждом шаге: паритет для не-тронутых раскладок, classic_ui цел,
  golden normalize цел (новые ключи presence-minimal).

## 6. Открытые вопросы владельцу (шаг 0) — ✅ РЕШЕНЫ 2026-07-23

1. Принципы П1–П5 — **ДА** (интерактивный hero = дефолт новых/демо;
   существующие сайты не трогаем, в билдере выбор).
2. Волна 1 = friseur + hotel + butcher/bakery — **ОК**.
3. G2 (выбор мастера при записи) — **ДА, в Волне 1**.
4. Restaurant — **два равных CTA** (Karte / Tisch reservieren).
5. Промо-архетип: авто-раскладка «акции первым экраном» — **ОК**.
