# Упрощение продукта «анти-Битрикс v2» — аудит + план (2026-07-08)

Задача владельца: функционал разросся, «похоже на Битрикс». Максимально упростить
для пользователя **создание / настройку / ведение** бизнеса. Скрывать нерелевантное
по архетипу, переносить ввод данных на витрину, пошаговый мастер, простой визуал.
Ниже — аудит по 4 осям (4 Explore-агента) + план. **Обсуждается, не начато.**

## 1. Диагноз — почему «Битрикс»
Причина не в количестве фич, а в том, что **слишком много видно СРАЗУ и одинаково у всех**:

- **15 пунктов меню всегда включены** независимо от архетипа: `dashboard` + `board` +
  **5 пунктов `catalog`** (Sortiment/Kategorien/Kombi/Import/**Lager**) + **10 пунктов
  `settings`** (Website/Einstellungen/Benachrichtigungen/Sprachen/Rechtstexte/
  Zusatzleistungen/Medien/Domains/Funktionen/Hilfe) + `billing`. `core=True` **слеп к
  архетипу** (`modules.py`).
- **`catalog` — это `core`** → товары/категории/Kombi/CSV-импорт/склад форсятся у
  парикмахера, отеля, Handwerker — у кого товаров нет вообще. `stock`/Lager сидит
  ВНУТРИ catalog → отдельно не выключить.
- **`settings` — один монолитный core-модуль на 10 пунктов** → нельзя убрать редкое
  (Domains/Medien/Sprachen/Rechtstexte) в «продвинутый» ящик без разбиения модуля.
- **`reviews`/`inbox`/`blog`/`gift` рекомендованы ВСЕМ 10 типам** → одинаковый шум.
- **«other» — свалка:** Friseur, Werkstatt, Handwerker, Retreat, promo — ВСЕ
  `business_type="other"`. От типа бизнеса они не получают полезного пресета —
  различие держится вручную на демо-китах (`enable_modules`). Нет типов
  Friseur/Handwerker/Werkstatt/Events + `jobs`/`events` вообще off-by-default.

**Хорошая новость (рычаги уже есть):** НЕ-core модули уже уважают `recommended_for`
per-архетип, а `ModuleGatingMiddleware` 404-ит скрытые URL. Механизм скрытия готов —
надо лишь распространить его на core + нарезать пункты. Инлайн-редактор витрины уже
правит тексты/имена/цены/даты/фото 5 сущностей + C-блоки.

## 3. Что уже правится с витрины vs только в кабинете (агент C)
- **Инлайн уже есть:** тексты (hero/about/cta/footer/каталог/корзина + заголовки
  секций), rename категории, у 5 сущностей — name/description/цена/дата/фото, C-блоки,
  раскладка.
- **Только кабинет:** СОЗДАНИЕ любой сущности (товар/услуга/номер/событие/акция/
  категория), структурные настройки (контакты/часы/право/склад/доставка/оплаты/SEO).
- **Топ-переносы на витрину (value/effort):** ①«+ добавить» в пустой секции (инфра
  инсертера уже есть) → растить каталог/услуги с сайта · ② инлайн-создание категории ·
  ③ `discount_percent` на карточку (тривиально — поле уже в реестре) · ④ бейдж
  остатка на карточке · ⑤ контакты/часы/Impressum инлайн (убирает главную причину
  лезть в settings).

## 4. Онбординг (агент D)
7 шагов (цель была ≤5); тип бизнеса спрашивается дважды (регистрация + шаг 1); **два
индикатора прогресса** на дашборде (Setup N/7 vs %-готовности — путает); «обрыв» до
мастера (ожидание провижининга ~1 мин → ре-логин на субдомене); право (Impressum)
ведётся слабо — галочка «готово» за один `address` (ложное соответствие). Демо-старт
«с примерами» и чек-лист готовности (архетип-aware) — работают.

## 5. План — 4 рычага
### L1. Per-архетип админка (снять «15 всегда-вкл»)  ← самый сильный де-Битрикс
- Дать core-модулям `recommended_for`/`hidden_for` + **гейт per-NavItem** → у сервис-
  архетипов убрать из меню весь `catalog` (или хотя бы Kombi/Import/Lager) и технические
  пункты `settings` (Domains/Medien/Sprachen/Rechtstexte) в «Erweitert»-ящик.
- Ввести реальные архетипы `friseur/handwerker/werkstatt/events` (миграция business_type
  ИЛИ отдельное поле `archetype`) → чистые пресеты вместо «other»-свалки.
- В «Простом» режиме дефолт reviews/inbox/blog — OFF.

### L2. Витрина-first ввод данных
- «+ добавить» в секциях (создать заглушку → заполнить инлайн уже готовыми хуками).
- Инлайн: категория-create, discount_percent, остаток, контакты/часы/Impressum.

### L3. Простой мастер
- ≤5 шагов, не переспрашивать тип, дефолт «с примерами», один индикатор прогресса,
  ведомое право, убрать обрыв (после регистрации сразу в мастер, а не ре-логин-лабиринт).

### L4. Визуал = простой (режим «Einfach/Experte» на весь кабинет)
- Как в редакторе: Einfach = минимум меню (Übersicht · Verkaufen[primary] · Website ·
  Einstellungen-lite); Experte = полный набор. Один тумблер.

## 6. Рекомендуемый порядок
**L1 (архетип-админка)** — прямой удар по «15 всегда-вкл», переиспользует готовый
`is_module_active`/`grouped_active_modules`. → **L2** (витрина-first create). → **L3**
(мастер). → **L4** (Einfach/Experte) — либо L4 первым как «зонтик», если хотим быстрый
визуальный эффект. Развилки — см. чат.

## 7. РЕШЕНИЕ ВЛАДЕЛЬЦА (2026-07-08) + консолидация
Владелец: **режим «Простой/Эксперт» на весь кабинет** (зонтик); модули **остаются
переключаемыми на «Funktionen»** (не жёстко прятать); реальные архетипы — **вводим
(миграция)**. Главное — **КОНСОЛИДАЦИЯ**: собирать связанное под один вход с табами,
длинные страницы — под табы, объединять «как общее» и вести пошагово.

**Целевая карта меню: ~25 плоских пунктов → ~7 хабов (Простой-режим).**

| Хаб | Табы/содержимое (сворачиваем сюда) | Из чего |
|---|---|---|
| **Übersicht** | дашборд + чек-лист готовности | dashboard |
| **Verkäufe** (доска-центр) | Board + табы Bestellungen/Termine/Übernachtungen/Tickets/Aufträge; карточка → деталь заказа / CRM | board + orders + booking + stays + events + jobs |
| **Sortiment** | табы Produkte / Kategorien / **Lager** / Kombi / Import | 5 пунктов catalog → 1 |
| **Marketing** | табы Aktionen / Gutscheine / Treuepunkte / Kampagnen / Bewertungen (+ Reservierungen/Einlösen под-таб) | promotions(3)+loyalty(2)+crm-кампании+reviews |
| **Kunden** | клиенты + Nachrichten(inbox) + рассылки | crm + inbox |
| **Website** | уже хаб: Preview/Home/Menü/Sections/Pages/SEO | settings-подстраницы |
| **Einstellungen** | табы Kontakt&Zeiten / Recht / Zahlungen / Versand / Benachrichtigungen · **Erweitert**-ящик: Sprachen/Medien/Domains/Funktionen/Abrechnung/Hilfe | settings(10)→1+ящик |

**КЛЮЧЕВОЙ ИНСАЙТ:** доска U-D **уже объединяет все типы транзакций** (заказы/брони/
билеты/заявки) → отдельные top-level списки Bestellungen/Termine/Übernachtungen/
Aufträge **дублируют доску** → сворачиваем их в табы/фильтры доски. −5 пунктов сразу,
почти бесплатно.

**Что ещё упростить в восприятии (мои добавления):**
- **Длинные портянки → табы:** `settings.html` (10 секций), форма товара, форма акции.
- **Objединить «как общее»:** «Verkäufe» = единый экран продаж (доска = операционный
  центр, per-тип — фильтр); «Marketing» = единый экран акций/лояльности.
- **Пошагово:** внутри хаба — табы; в первичной настройке — шаги мастера (L3).
- **Простой-режим прячет нерелевантные хабы по архетипу:** Friseur → нет «Sortiment»
  (или только Lager off), «Verkäufe» показывает лишь таб Termine; Hotel → нет корзины.
- **Эксперт-режим** = все хабы/табы развёрнуты (или плоский полный список — как сейчас).

**Порядок реализации (черновой):** (0) поле/миграция `archetype` + Простой/Эксперт-флаг →
(1) паттерн «хаб+табы» на ОДНОМ хабе как эталон (кандидат: **Sortiment** — чистый 5→1) →
(2) Verkäufe (свести списки в доску) → (3) Einstellungen-табы+Erweitert → (4) Marketing/
Kunden → (5) Простой-режим прячет хабы по архетипу → (6) L2 витрина-first → (7) L3 мастер.
