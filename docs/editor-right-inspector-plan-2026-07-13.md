# Редактор: правый инспектор вместо ленты/листа (2026-07-13)

Решение владельца: «A + мелочи из B» (обсуждение 2026-07-13). Диагноз: хром браузера
(~110px) + тулбар (~40px) + раскрытая лента/лист настроек (80–120px+) = треть экрана
на ноутбуке. Вертикальный пиксель — самый дорогой, горизонтальный — самый дешёвый.

## Целевая компоновка

1. **Верхний тулбар — одна тонкая строка (как сейчас), компактнее:** подписи кнопок
   ☰/🧱/⚙️/🔗 скрываются раньше (иконки+title), девайс-переключатель — только иконки
   до xl. Высота ~40px всегда.
2. **Один правый инспектор** (`#bld-editor-pane`, те же id → JS-логика цела):
   - десктоп (lg+): `fixed right-0 top-40px bottom-0 w-[360px]`, вертикальный скролл,
     `border-l shadow-xl`; вместо полноширинного листа под тулбаром;
   - контент: те же области (вкладки Theme/Banner/Menu/Footer/Library) ИЛИ настройки
     выбранного блока (лента) — теперь вертикальным стеком (поля UC6-9 уже
     label-сверху/контрол-снизу, переезжают 1:1);
   - мобайл (<lg): нижний лист (как было у ленты) — max-h-[60vh] bottom-sheet.
3. **Канва сжимается, не перекрывается:** при открытом инспекторе у канвас-области
   `margin-right: 360px` (lg+); после toggle — вызвать `applyDevice(currentDev)`
   (пересчёт десктоп-масштаба; сейчас он пересчитывается только на resize).
4. **Мелочи из B:** Esc закрывает панель (есть); двойной клик по пустому месту канвы
   закрывает; состояние «свёрнуто» — в localStorage (есть у листа — сохранить).

## Что НЕ трогаем

- Всю JS-логику областей/попапа блока (showArea, openBlockPopup, collect, draft,
  swapPreview) — меняется ТОЛЬКО геометрия контейнера (CSS-классы) + один вызов
  applyDevice в toggle-путях.
- Мобильное поведение (bottom-sheet) — переносится классами, не логикой.
- id элементов — все сохраняются (замки test_home_builder ссылаются на id).

## Риски

- applyDevice масштабирует по `pane.clientWidth` (родитель канвы): после сжатия
  канвы пересчитать; при закрытии — тоже. Забыть = «десктоп» с неверным масштабом
  до первого resize.
- Лента открывалась классом `bld-ribbon-open` с `display:block!important` (фикс W1
  HIGH-регрессии) — семантика «показать панель при выбранном блоке» сохраняется,
  меняются только габариты.
- Инспектор поверх fixed-элементов витрины в канве — канва в iframe, z-конфликтов нет.

## Проверка (стенд Playwright)

1. клик по блоку → панель справа, канва сжата, поля блока видны и работают
   (live-своп жив); 2. «⚙️ Template» → та же панель с вкладками областей; 3. ✕/Esc →
   канва во всю ширину, масштаб пересчитан; 4. девайс-переключатель при открытой
   панели; 5. мобильный вьюпорт → нижний лист; 6. undo/redo/Save как раньше.
