# UC2-3 — инспектор-группы всех page_type (план до кода, 2026-07-02)

Инкремент волны U-C (`uc-plan §11` п.6). Спека: «applyPageScope/SCOPE_PAGE_KEY →
реестр-driven; каждый page-type переключателя получает инспектор-группу; C-блоки на
деталь/инфо». Анализ против кода (после UC2-1/UC2-2).

## 1. Что УЖЕ есть (не переделывать)

- `SCOPE_PAGE_KEY` (site_home.html:1751) покрывает 4 листинга + 4 детали + cart —
  все СУЩЕСТВУЮЩИЕ инспектор-группы `.page-block[data-page-key]` уже скоупятся.
- Драфт per-page ключей generic (UC2-1 `apply_page_payload`); клик→инспектор на
  всех деталях (UC2-2 слайс 1).

## 2. Реальные пробелы UC2-3

- **(a) Хардкод-карта → реестр.** `SCOPE_PAGE_KEY` дублирует знание реестра.
  Генерировать из `siteconfig.PAGE_CONFIG_KEYS`/`page_types()` (шаблонный JSON в
  `site_home.html` из view-контекста). S, без рисков.
- **(b) info/legal не управляются конфигом.** Их секции — фикс-реестр (UC1-2:
  «управление — UC2-3/UC3-2»). Дать hide-управление = НОВЫЕ config-ключи
  (`info_page`/`legal_page` `{hidden:[…]}` по образцу `*_detail`) + расширение
  normalize (⚠️ горячее, golden-замки) + гейт в шаблонах info/legal + page-block
  в билдере + collect()/draft (через `apply_page_payload` — одна строка в
  реестре). M. Ценность средняя (секций мало: about; impressum/datenschutz/
  widerruf скрывать юридически опасно — право должно быть видимым!). ⚠️ Вывод:
  legal НЕ делать hide-управляемым (DACH-риск); info (about) — можно, но это
  одна секция. Кандидат на «не делать» — решение владельца.
- **(c) C-блоки на деталь/инфо** — ЗАБЛОКИРОВАНО тем же решением per-page
  хранения секций, что слайс 3 UC2-2 (`config["sections"]` — home-only;
  см. `uc2-2-oncanvas-plan §2`). Единое решение владельца на оба пункта.

## 3. Предлагаемый объём UC2-3 (ревизия)

Сделать (a) — реестр-driven карта скоупов (убирает дубль знания, готовит почву
для новых page_type). НЕ делать (b) для legal (право прятать нельзя); (b) для
info отложить как микро-ценность; (c) — ждёт решения владельца о per-page
секциях (вместе со слайсом 3 UC2-2 и UE1-хостом — ОДНО архитектурное решение).

**РЕШЕНИЕ ВЛАДЕЛЬЦА (2026-07-03): per-page хост C-блоков ПОКА НЕ ДЕЛАЕМ.**
Пакет UC2-3(b)+UC3-2+слайс 3 UC2-2 остаётся отложенным (см. roadmap §Отложено);
вернуться можно этим же план-доком, когда владелец захочет блоки вне главной.
