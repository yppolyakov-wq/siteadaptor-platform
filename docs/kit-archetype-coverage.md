## 2. По архетипам: база направления ↔ наше ↔ демо ↔ что добавить

### A4 — Gastro (restaurant, pranasy) — покрытие ~95 %
- **База направления:** меню с фото/категориями, аллергены (LMIV), Extras/
  модификаторы, комбо, онлайн-заказ (Abholung+Lieferung), бронь стола, QR-заказ
  за столом, акции, лояльность, события, часы, отзывы.
- **Наше:** всё есть — catalog+modifiers (A4a/b), combos, orders C&C+Versand (G4),
  booking-стол (D3), QR-Bestellung (T2a), KDS, промокоды (Voucher), loyalty, events.
- **Демо показывает:** варианты, модификаторы, аллергены, доставку, акции, события.
- **Что добавить (мелочи):**
  - В демо нет **комбо-набора** (Combo) и **дайн-ин QR-стола** — фичи есть, в
    showcase не видны. → добавить 1 комбо + лист QR-столов в restaurant/pranasy.
  - Бронь стола: ресурс есть, но в демо нет ни одной **брони стола** в кабинете
    (seed_records льёт заказы/события, но не booking-брони для gastro). → засеять.
  - **Депозит за стол** / no-show — не показан.

### A5 — Übernachtung (hotel) — покрытие ~90 %
- **База:** типы номеров с фото/описанием, календарь доступности, сезонные цены,
  депозит/предоплата, iCal-синк каналов, удобства/завтрак, карта, отзывы, FAQ.
- **Наше:** units+фото (новое), availability, SeasonRate (A5a), депозит (E4),
  iCal экспорт/импорт (A5b), авто-Rechnung (A5c).
- **Демо показывает:** 4 номера с фото/описанием. **Не показывает:** сезонные
  тарифы, депозит, iCal, завтрак-как-extra, бронь в кабинете.
- **Что добавить:**
  - Засеять в hotel-демо: 1 **сезонный тариф**, **депозит** на номер, пару
    **броней** в кабинет, и FAQ/Trust секции (сейчас пусто).
  - **Завтрак/extras** к брони (нет механики extras у stays) — отдельная фича.
  - **Карта** местоположения на странице номера (гео есть в агрегаторе, на
    витрине stays — нет).

### A1/A2 — Retail Online-Shop — покрытие движка ~100 %, демо ~30 % ⚠
- **База:** каталог с **вариантами** (размер/вес), **Grundpreis** (€/кг, PAngV),
  **остаток** (anti-oversell), GTIN/EAN, фото-галерея, категории/фильтр, корзина,
  Abholung+**Versand с зонами по PLZ**, предоплата, product-feed, возвраты.
- **Наше:** всё есть — R1 варианты, R2 Grundpreis, R3 остаток, A1 GTIN, A2 Versand
  +зоны+Lieferschein+Widerruf, Google-feed.
- **Демо:** ❌ **главный пробел.** aktionsmarkt = промо-витрина (16 товаров без
  вариантов/Grundpreis/остатка/доставки). Ни один кит не показывает R1–R3 + A2.
- **Что добавить (приоритет #1):**
  - **Новый кит «Retail Online-Shop»** (Hofladen/Boutique, business_type=retail):
    товары с вариантами+Grundpreis+остатком, GTIN, доставка с PLZ-зонами,
    Mindestbestellwert, возвраты, наполненный кабинет (заказы/возвраты). Это
    делает видимым большой пласт уже готового кода.

### A3 — Termin-Dienstleister (friseur) — покрытие ~90 %
- **База:** услуги (цена+длительность), мастера/ресурсы, онлайн-запись,
  доступность, депозит, **Mehrfachkarte/абонемент**, лояльность, отзывы.
- **Наше:** services (G10), resources, availability, депозит, **Pass/PassPlan**
  (G9b + A3 онлайн-продажа), loyalty.
- **Демо:** 6 услуг + 2 ресурса + лояльность. **Не показывает:** Mehrfachkarte
  (Pass), депозит, выбор конкретного мастера.
- **Что добавить:**
  - Засеять **PassPlan** (10er-Karte) + 1 выданную карту в friseur-демо.
  - **Выбор конкретного мастера** на витрине: сейчас движок берёт первый
    свободный ресурс; для парикмахерской/массажа клиент часто хочет «к Лене». →
    опц. выбор ресурса в `/termin/` (фича, не только демо).
  - Ресурсы как `staff`-тип (сейчас «Stuhl»=table).

### A6 — Event/Retreat (retreat) — покрытие ~85 %
- **База:** события с вместимостью, **ценовые категории билетов** (Frühbucher/
  Einzel/Doppel), Programm, анкета, ростер, оплата, лендинг, отзывы.
- **Наше:** events (A6) + ретрит-лендинг (новое) + 1:1-booking + merch-каталог.
- **Демо:** богатый ретрит-лендинг, 4 события.
- **Что добавить:**
  - **Ценовые тиры билета** (сейчас одна `price_cents` на событие) — Frühbucher/
    Standard/Kind. Реальная дыра A6.
  - Связка **событие ↔ проживание** (retreat с ночёвкой: events + stays).

### A7/A9 — Handwerker/Werkstatt (werkstatt) — покрытие ~100 %
- **База:** заявка с фото, Angebot/Kostenvoranschlag, Auftrag→Rechnung, Fahrzeug,
  запись на приём, расходники из каталога со списанием склада.
- **Наше:** jobs (A7), фото (A7b), депозит (A7c), Termin-связка (A7d), Fahrzeug
  (A9), расходники catalog→stock (G11).
- **Демо:** услуги + сметы (Fahrzeug-Angebote) + каталог Teile.
- **Что добавить:**
  - Засеять **смету с расходником из каталога** (G11) — симбиоз не виден в демо.
  - Опц. кит «Handwerker» (Maler) — generic, без авто-специфики (низкий приоритет).

## 3. Сквозные usability-выводы

1. **Демо отстаёт от движка.** Много готовых фич не видны ни в одном showcase:
   варианты/Grundpreis/остаток (retail), комбо, депозиты, сезонные тарифы, iCal,
   Mehrfachkarte, ценовые тиры билетов, расходники сметы. → завести их в
   соответствующие киты + **новый retail-кит**.
2. **customer_account (ЛК клиента)** выключен по умолчанию у типа `other` →
   friseur/werkstatt/retreat демо его не показывают. → добавить в их
   `enable_modules`, если хотим демонстрировать ЛК.
3. **inbox («Frage stellen»)** и **Telegram** не включены ни в одном ките.
4. **Отзывы** живут в агрегаторе; на витрине бизнеса отзывов нет (только
   авто-рейтинг в Trust). Для доверия (отель/ретрит/салон) — рассмотреть блок
   отзывов на самой витрине.
