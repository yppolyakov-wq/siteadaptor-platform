# Концепт «SiteAdaptor Studio»: визуальный конструктор нового поколения + лёгкая админка

> Создан 2026-07-18 по запросу владельца: «расширять шаблоны каждого блока/страницы
> с привязкой к архетипу; конструктор, которым соберёт сайт человек без знаний;
> админка лёгкая, концентрируется на канбане/товаре/маркетинге/интеграциях/настройках;
> уйти от списков к визуализации; быть Apple в конструкторах для малого бизнеса —
> функции китов, интуитивность выше». Статус: КОНЦЕПТ НА ОБСУЖДЕНИЕ (код — после
> решений владельца §8). ID: **ST** (Studio) — внести в task-catalog при старте кода.
> Мокапы: артефакт «SiteAdaptor Studio — концепт» (интерактив, 3 экрана).

## 1. Тезис

Мы НЕ делаем ещё один Wix (свободная канва пугает новичка) и не Jimdo (жёсткий
шаблон скучен и «не мой»). Наш тренд — **«сайт собирается выбором, а не
дизайном»**: на каждом уровне (образ сайта → страница → блок) пользователь
ВЫБИРАЕТ из 3–5 готовых вариантов, персонализированных под ЕГО архетип, и сразу
видит СВОЙ сайт (реальные товары/услуги/фото/часы, демо-рыба там, где пусто).
Пустой страницы и «дизайнерских решений» не существует. Это уже наша ДНК
(решения анти-Битрикс 2026-06-26…07-13) — концепт доводит её до цельного
продукта с единым визуальным языком.

Формула: **Выбор вместо дизайна · Твой контент вместо lorem · Один экран вместо
меню · Живое превью всегда.**

## 2. Рынок (веб-рисёрч 07.2026; полный отчёт — в чате сессии 2026-07-18)

- **Планка онбординга 2026** — Durable (3 вопроса → сайт за 30 сек), Jimdo
  Dolphin (~3 мин, DE, авто-Impressum/AGB), Wix AI-чат (~60 сек, ADI выключен
  11.2024), Squarespace Blueprint (5 гайд-шагов, >50 % новых клиентов), IONOS
  (3–8 вариантов дизайна на выбор), GoDaddy Airo (агентный чат). Нас будут
  сравнивать с «30 сек — 3 мин до живого сайта».
- **Section-based побеждает freeform** для нетехничных (Squarespace Fluid Engine
  vs Wix-наложения): галереи готовых секций-вариантов — норма. Наша ставка
  «выбор вместо дизайна» = мейнстрим-валидна, мы её только доводим до конца.
- **«Design intelligence»** (Squarespace Site Themes: курируемые палитры/
  шрифт-паки; Layout Switcher: перекладка секции в проф-layout одним кликом) —
  прямой аналог нашего L1 Look/L2 Page. Никто не даёт 3–5 согласованных
  пресетов ПОД АРХЕТИП с реальным контентом — наше отличие.
- **Визуал 2026**: bento-grid, крупная типографика, сдержанный glassmorphism
  для глубины (не декор; blur дорог на Android); двойное движение — AI-фичи +
  уход от «генерик-AI-вида» к человеческой фотографии/типографике.
- **Админки SMB**: Shopify ~7 пунктов (Orders/Products/Marketing/…); Square —
  единая лента заказов всех каналов + разговорный Square AI; Fresha/Shore/
  Treatwell (услуги) — центр = **КАЛЕНДАРЬ записей**, не канбан. Вывод для нас:
  визуализация «сегодня» должна переключаться Канбан ⇄ Календарь ⇄ Лента, и
  дефолт зависит от архетипа (салон/мастерская — календарь, магазин — лента,
  смешанный — канбан). Рынок держит 5–8 пунктов меню.
- **Маркетинг, который офлайн-SMB реально использует**: №1 по ROI —
  авто-напоминания SMS/email/**WhatsApp** (Shore: −55 % no-shows), затем
  лояльность по телефону на кассе (Square Loyalty), локальные отзывы/Google —
  а НЕ «рассылки». Наш Marketing-хаб должен вести с напоминаний/лояльности/
  отзывов, кампании — вторым слоем. DACH-плюс: double-opt-in у нас уже
  first-class (UWG §7), у US-тулов — нет.
- **Дыра рынка (подтверждена)**: никто не связывает сайт+запись+заказ+маркетинг
  под DACH-правом в одном простом продукте. Durable/Square — all-in-one, но
  US-центричны (DSGVO/Impressum слабые); Jimdo/IONOS — только сайт; Shore/
  Timify/Fresha — только запись (и beauty-only агрегаторы). Формула ниши:
  «Durable-класс all-in-one + Squarespace-простота + DSGVO/Impressum по
  умолчанию + WhatsApp + мультиязык + локальный агрегатор — за 39 €/мес».
- Вывод: конкурируем НЕ шаблонной красотой (там паритет), а тем, что шаблон
  живёт поверх РЕАЛЬНОЙ операционки бизнеса и его контента, под местным правом.

## 3. Система шаблонов: 3 уровня × 14 архетипов

Вся инфраструктура НИЗА уже построена — концепт добавляет ВЕРХ и СВЯЗНОСТЬ:

| Уровень | Что выбирает владелец | Что уже есть (реестр) | Что добавляем |
|---|---|---|---|
| **L1 Look («образ»)** | весь стиль сайта одним кликом: палитра, шрифт, стиль карточек, hero, набор секций | `sitetemplates.TEMPLATES` (8 универсальных) | **3–4 Look'а НА КАЖДЫЙ архетип** (≈45 шт.: «Bäckerei · Warm Craft», «· Modern Dark», «· Minimal»…), каждый = тема+шрифт+card style+hero style+section set+фото-настроение демо; `recommended_for` уже есть |
| **L2 Page (страница)** | раскладка конкретной страницы | главная: `page_templates`+`SECTIONS`; категория: `LAYOUT_PRESETS` (5); деталь: пресеты карточек+секций (AB6.10); «О нас»: 4 пресета블 (AB6.10); шапка: `NAV_STYLES` (3) | довести до ВСЕХ типов страниц реестра `page_types()` (меню/контакт/корзина/тексты), по 3–5 вариантов, фильтр по архетипу; листинги услуг/номеров/событий — те же пресеты, что каталог |
| **L3 Block (блок/секция)** | вид отдельного блока | `CBLOCK_VARIANTS` (до 10/тип), `SECTION_STYLES` (5 секций × 4–5), `MICRO_TEMPLATES` (5) | наполнить пустые слоты «10 видов на блок» (реестры готовы, фидбэк UC6) + архетипные варианты (карточка товара пекарни ≠ карточки услуг салона) |

**Ключевое отличие от рынка — «живая галерея»**: каждый вариант в галерее
рендерится РЕАЛЬНЫМ контентом тенанта в миниатюре (scaled-iframe — механика уже
есть в превью мастера). Шаблон показывает «как будет выглядеть ТВОЙ сайт», а не
чужую картинку. Ни у Wix, ни у Squarespace этого нет (их галереи — статичные
превью с чужим контентом).

Хранение: L1 — материализация в site_config через `apply_template` (расширить
поля темы); L2/L3 — существующие ключи (`page_templates`, `catalog_layout`,
`<module>_detail`, `page_blocks`, `sections`). **Новых top-level ключей normalize
не требуется** (риск golden-замков — нулевой в v1).

## 4. Studio — конструктор (переупаковка, не переписывание)

Редактор уже умеет главное (канва, click-to-edit, drag, инсертер, live-превью
без перезагрузки, right-inspector, ribbon). Studio = новый ХРОМ поверх:

1. **Полноэкранная оболочка** (без хрома кабинета): слева вертикальная рейка
   уровней — «🎨 Look · 📄 Seiten · 🧱 Blöcke · 🖼 Medien»; по центру канва
   (существующая); справа инспектор (существующий).
2. **Look-переключение с кросс-фейдом**: клик по Look'у в галерее → канва
   мягко перерисовывается (механика двойной буферизации уже есть — своп кадров).
3. **Page-галерея**: выбор страницы сверху (уже есть `?page=`-скоуп) → лента
   вариантов её раскладки снизу (как выбор трека в плеере), апплай в один клик,
   Undo сохраняется (история уже есть).
4. **Медиа**: единая библиотека фото тенанта + демо-фонд (298 ключей) + (v2)
   AI-генерация. Замена фото = клик, как сейчас 📷.
5. **Мобайл-превью** — таб device (уже есть applyDevice).

Правило Studio: **на экране не больше одного решения за раз** (уровень → выбор →
превью). Никаких форм с 20 полями — формы остаются в кабинете «Эксперт».

## 5. Админка: 5 хабов + визуальный хоум

Сайдбар-СПИСОК умирает. Главная кабинета = **приборная панель**:

1. **📥 Заказы** — переключатель **Канбан ⇄ Календарь ⇄ Лента** (UD2-доска и
   календари уже есть), дефолт по архетипу (рыночный паттерн: услуги живут в
   календаре — Fresha/Shore; магазин — в ленте); бейджи «сегодня».
2. **🏷 Angebot** — товары/услуги/номера/события ГРИДОМ КАРТОЧЕК с фото
   (экран FB-8 уже объединил сущности; таблицу → в карточки), тумблер видимости
   на карточке. Поглощает FB-8-развитие.
3. **📣 Marketing** — витринная фишка. Порядок ПО ROI офлайн-SMB (рисёрч):
   (1) авто-напоминания записей/заказов (email/Telegram сейчас; WhatsApp —
   external-backlog), (2) лояльность/штампы, (3) отзывы+Google-профиль,
   (4) акции→все каналы одной кнопкой (сайт/агрегатор/соцсети/email/Telegram),
   (5) кампании по сегментам. Всё построено (S4a) — довести до «одна кнопка»
   + панель результатов (показы/клики/погашения).
4. **🔌 Integrationen** — платежи, Telegram, OAuth-каналы, домены, вебхуки, OTA.
5. **⚙️ Einstellungen** — как сейчас (W4 аккордеоны), + «Website bearbeiten» →
   Studio.

Хоум = 5 плиток-хабов с живыми числами (плитки AB7 уже есть — переработать в
виджеты: выручка-спарклайн, заказы-сегодня, «маркетинг-пульс») + встроенный
канбан. Простой/Эксперт остаётся; ⌘K-поиск по действиям (v2).

**Визуальный язык** (и Studio, и админка): системная типографика (SF-стек),
крупные заголовки, карточки с мягкой тенью вместо таблиц, много воздуха, ОДНА
акцентная кнопка на экран, единый иконсет (решение §8.3), тёмная тема, motion
только функциональный (кросс-фейд превью, snap канбана). Числа — большие,
таблицы — только в «Эксперте».

## 6. Онбординг (уже почти готов)

Регистрация (double-opt-in AB5.1) → мастер AB6.10 = тот же выбор вариантами.
Изменение по концепту: слайд «Stil» становится слайдом **«Look»** (галерея
архетипных Look'ов L1 вместо 8 универсальных) — дальше мастер уже показывает
живое превью. Опционально v2: 3 вопроса → авто-подбор Look'а («AI-онбординг»
без AI-затрат: правила по архетипу+городу+фото).

## 9. Риски

- Look'и умножают поверхность демо/тестов → golden-замок «каждый Look проходит
  apply_template+normalize» (по образцу CBLOCK_VARIANTS-замка).
- Живые мини-превью галереи = N iframe'ов → рендерить лениво (IntersectionObserver),
  кэшировать статикой (v2 — скриншот-сервис).
- Редизайн админ-хоума не должен ломать привычку — Эксперт-режим и старые URL
  живы (гейт-принцип W0: скрытие только CSS/навигацией, не удаление).
- Шрифты Look'ов — только self-hosted/системные (CSP, GDPR — без Google Fonts CDN).
